# 规范输出规则

> 隶属：周道语言规范 1.1 — 010.5 终态  
> Formatter 三模式：preserve（默认）保留用户原写法，verbal/symbolic 按此处输出。  
> 本文件不限制用户源码书写风格。

---

## 1. 标点规则

| 规则 | 规范形式 | 非法形式 |
|------|---------|---------|
| 句子结束 | `。` | 无句号 |
| 逗号分隔 | `，` | 半角 `,` |
| 参数分隔 | `、` 或 `，` | 混用 |
| 字符串 | `【内容】` | 引号 `"内容"` |
| 精确名称 | `{名称}` | `｛名称｝` （允许但不推荐） |
| 模块名 | `《os》` | 无书名号 |

## 2. 语句格式

| 语句 | 规范形式 |
|------|---------|
| 绑定 | `设甲为乙。` |
| 变更 | `使甲变为乙。` |
| 算术变更 | `使甲加乙。` |
| 打印 | `显示甲。` |
| 函数定义（单表达式） | `设甲（乙）为丙。` |
| 函数定义（多语句） | `定义甲（乙）如下：动作。` |
| 类别方法定义 | `定义甲类别的乙（丙）如下：动作。` |
| 类别声明 | `设置甲类别，包括乙、丙。` |
| 引入 Python 模块 | `引入Python模块《甲》。` |
| 从 Python 模块引入 | `从Python模块《甲》中引入乙。` |
| 引入周道源文件 | `引入周道源文件《甲》。` |
| 从周道源文件引入 | `从周道源文件《甲》中引入乙。` |
| 模块接口 | `规定模块接口：甲、乙。` |
| 运行入口 | `运行如下：动作。` |
| 报错（无类型） | `报错【消息】。` |
| 报错（指定类型） | `报错【消息】，错误类型是值出错。` |
| 原样重抛 | `原样报出当前错误。` |
| 空操作 | `不作处理。` |
| 断言 | `甲须大于0。` |
| 全局声明 | `下文所用甲，均指全局的。` |
| 外层声明 | `下文所用甲，指本定义外层的。` |

## 3. 表达式格式

| 表达式 | 规范形式 |
|--------|---------|
| 列表 | `［甲、乙］` |
| 固定序列 | `固定序列［甲、乙］` |
| 集合 | `集合［甲、乙］` |
| 映射 | `［键为值］` |
| 字符串下标 | `甲【键】` |
| 表达式下标 | `甲［索引］` |
| 切片 | `甲［开始：结束］` |
| 成员访问 | `甲的乙` |
| 函数调用 | `甲（乙）` |
| 制定参数 | `甲（名称为值）` |

## 4. 结构规则

1. `运行如下` 必须位于文章最后
2. 模块接口声明必须位于 `运行如下` 之前
3. 同一模块至多一个 `运行如下`
4. 同级多语句使用分行或分号，不使用逗号代替语句分隔
5. 定义体缩进一个层级（4 空格）
