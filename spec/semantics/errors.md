# 错误处理语义规范

> 隶属：周道语言规范 1.0 — 第三层：语义规则

---

## 1. 错误类型映射

| 周道错误名 | Python 异常类 | 首次版本 |
|-----------|--------------|---------|
| 运行出错 | RuntimeError | 004（默认） |
| 值出错 | ValueError | 009 |
| 类型出错 | TypeError | 009 |
| 键出错 | KeyError | 009 |
| 索引出错 | IndexError | 009 |
| 文件未找到出错 | FileNotFoundError | 009 |

## 2. 报错

### 2.1 无类型报错

```
报错【消息】。
```

生成 `raise RuntimeError("消息")`。行为与 004 冻结一致。

### 2.2 指定类型报错

```
报错【消息】，错误类型是值出错。
```

生成 `raise ValueError("消息")`。错误类型必须是已知错误类型符号。

## 3. 分类捕获

```
尝试 body；
如果错误类型是类型1，就 handler1；
如果错误类型是类型2，就 handler2；
如果出错，就 default_handler。
```

### 3.1 规则

1. 分类处理按源码顺序匹配（first-match）
2. 重复捕获同一静态错误类型 → 诊断
3. `如果出错` 至多出现一次，必须位于所有分类之后
4. 错误类型必须是明确符号，不使用字符串比较

## 4. 原样重抛

```
原样报出当前错误。
```

### 4.1 规则

1. 只允许在 `如果错误类型是` 或 `如果出错` 分支中使用
2. 原样抛出当前捕获的错误，保留类型、消息和 Python 因果链
3. 不重新构造错误对象
4. finally 中已离开错误上下文时不得使用

## 5. 运行时异常映射

Python 异常 → 周道源码位置的映射规则：

1. 同步函数调用栈：每一帧映射到对应周道函数定义位置
2. 生成器暂停/恢复：保留生成器创建位置
3. 异步：生产端和消费端分别保留位置，事件循环帧隐藏
4. 跨模块：保留定义文件与调用文件身份
5. 第三方库内部异常保留原始类型和因果链
