# 源码结构与编码规范

> 隶属：周道语言规范 1.0 — 第三层：语义规则

---

## 1. 编码

| 属性 | 规则 |
|------|------|
| 编码 | UTF-8（无 BOM） |
| 换行 | LF（`\n`）为标准；CRLF（`\r\n`）兼容接受 |
| 文件末尾 | 必须有换行 |
| BOM | 不接受 UTF-8 BOM |

## 2. 缩进

| 属性 | 规则 |
|------|------|
| 缩进单位 | 4 个空格 |
| Tab | 不接受 Tab 缩进 |
| 混用 | 同一文件不得混用空格和 Tab |

缩进用于以下结构：
- `定义…如下：` 之后的定义体
- `如果…就` 之后的动作块
- `当…时，一直` 之后的循环体
- `尝试` 之后的分支体
- `运行如下：` 之后的入口体
- `分情形：` 之后的分支

## 3. 续行

长表达式可以在以下位置换行：

```
设结果为函数（
    参数一、
    参数二
）。
```

规则：
- 括号 `（` `）` 内允许自然换行
- 方括号 `［` `］` 内允许自然换行
- 字符串 `【` `】` 内允许自然换行
- 换行后应缩进一个层级

`；` 分隔的语句不可跨行。

## 4. 空行

- 空行不终止块结构
- 注释行不影响缩进层级
- 定义之间建议用空行分隔

## 5. 顶层语句与模块初始化

### 5.1 定义与声明

顶层定义的函数、类别、方法仅注册，不在引入时执行函数体。

### 5.2 顶层执行语句

顶层非定义语句会在模块引入时执行。

```
设缓存为映射［］。      ← 引入时执行
显示【已加载】。         ← 引入时执行
```

### 5.3 运行入口

`运行如下：` 的内容在直接执行时运行一次，被引入时不执行。

### 5.4 模块缓存

模块只初始化一次。首次引入后缓存结果，后续引入返回同一对象。
