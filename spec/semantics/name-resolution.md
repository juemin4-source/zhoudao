# 名称解析语义规范

> 隶属：周道语言规范 1.0 — 第三层：语义规则

---

## 1. 作用域

周道采用**词法作用域（静态作用域）**。

### 1.1 作用域类型

| 类型 | 创建时机 | 包含 |
|------|---------|------|
| 全局 | 模块开始 | 顶层绑定、函数名、类别名 |
| 函数 | 每次函数/方法定义 | 参数、函数内局部绑定 |
| 控制 | 每次 if/while/for/try/分情形 | 循环变量等临时绑定 |

### 1.2 嵌套规则

- 函数作用域嵌套在全局作用域内
- 控制作用域嵌套在函数或全局作用域内
- 子作用域可访问父作用域的名称
- 同一作用域内不同类型可以同名（如类别名和变量名）

## 2. 名称解析算法

```
解析名称(name):
  1. 从当前作用域开始向上查找
  2. 找到第一个匹配即返回
  3. 直达全局作用域仍未找到 → 查 PreludeScope
  4. PreludeScope 未找到 → 报"未定义的名称"
```

### 2.1 跨越声明

- `下文所用name，均指全局的` → 跳过当前函数层，直接到全局
- `下文所用name，指本定义外层的` → 跳过当前函数层，到外层函数

## 3. 精确名称

### 3.1 词法规则

精确名称从 `{` 到 `}` 构成单一 IDENTIFIER Token，`是否精确=True`。

```
{用户的姓名} => IDENTIFIER("用户的姓名", 是否精确=True)
```

### 3.2 语义规则

1. 精确名称跳过所有上下文关键字判断
2. 精确名称始终解析为单一 SymbolName，不具有对象-成员结构
3. `{用户的姓名}` 与 `用户。姓名` 没有隐含转换关系
4. `{自己}` 在类别方法中不绑定到 self，而是普通名称

### 3.3 对比

| 写法 | 解析结果 | 语义 |
|------|---------|------|
| `用户的姓名` | 成员访问(用户, "姓名") | 运行时读取成员 |
| `{用户的姓名}` | 变量引用("用户的姓名") | 名称引用 |
| `自己` | 变量引用("自己") | 方法中绑定到 self |
| `{自己}` | 变量引用("自己") | 普通名称，不绑定到 self |

## 4. 名称空间

| 名称来源 | 名称空间 | 是否参与周道名称解析 |
|---------|---------|-------------------|
| 用户定义变量 | 周道统一名称空间 | 是 |
| 函数名 | 周道统一名称空间 | 是 |
| 类别名 | 周道统一名称空间 | 是（可与变量同名的不同绑定） |
| 从周道源文件 import 的名称 | 周道统一名称空间 | 是 |
| Python 模块成员 | Python 模块名称空间 | 否（通过成员访问调用） |
| 从 Python 模块 import 的名称 | 周道统一名称空间 | 是 |
