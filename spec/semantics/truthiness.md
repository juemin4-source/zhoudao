# 条件真假性与逻辑运算符语义

> 隶属：周道语言规范 1.0 — 第三层：语义规则

---

## 1. 条件真假性

### 1.1 规则

周道继承 Python 的隐式真假性：

| 值 | 真/假 |
|------|--------|
| `成立` (True) | 真 |
| `不成立` (False) | 假 |
| `没有值` (None) | 假 |
| `0`、`0.0` | 假 |
| 空文本 `【】` | 假 |
| 空列表 `［］` | 假 |
| 空固定序列 `固定序列［］` | 假 |
| 空集合 `集合［］` | 假 |
| 空映射 `［］` | 假 |
| 其他值 | 真 |

### 1.2 诊断

当条件表达式为以下情况时，发出警告：

- 字面量常量（如 `如果成立`、`如果不成立`）
- 非布尔类型的变量，建议显式比较

```text
警告：条件「如果甲」使用了隐式真假性，建议写为「如果甲不等于【】」
```

该诊断级别由项目配置控制：

```toml
[diagnostics]
implicit_truthiness = "warning"  # "off" | "warning" | "error"
```

### 1.3 适用范围

此规则适用于：
- `如果` 条件
- `当` 循环条件
- `依` 分情形对象
- `就` 之间的动作条件

## 2. 逻辑运算符

### 2.1 且 / 或

`且` 和 `或` 用于条件上下文，返回布尔值 `成立` 或 `不成立`，不返回原对象。

| 表达式 | 结果 |
|--------|------|
| `真且真` | `成立` |
| `真且假` | `不成立` |
| `甲或乙` | 布尔值（非甲或乙本身） |

### 2.2 短路规则

`且` 和 `或` 遵循短路求值：

```
甲且乙：甲为假时，不计算乙
甲或乙：甲为真时，不计算乙
```

短路仅控制是否求值，不影响返回值类型。

### 2.3 与 Python 差异

Python：`0 or 42` → `42`
周道：`0或42` → `成立`

这是有意的设计选择。母语内语中"甲或乙"表达的是逻辑判断，不是对象择取。

---

## 3. 求值顺序

表达式求值继承 Python 的左到右规则：

```
函数甲（）加函数乙（）
```

先调用 `函数甲()`，再调用 `函数乙()`，然后相加。

指定参数：

```
处理（甲为函数一（）、乙为函数二（））
```

从左向右求值：先 `函数一()`，再 `函数二()`，然后绑定参数。
