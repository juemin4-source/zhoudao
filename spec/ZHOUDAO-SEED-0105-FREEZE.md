## 2. 冻结范围

### A — 表达式与使语句

| 能力 | 冻结行为 |
|------|---------|
| 双表层 | 汉语式 `甲加乙` 和数学符号 `甲 + 乙` 均为正式语法，语义等价，归一到同一 Core IR |
| 二元运算符 | `+ - * / ** // % = != < <= > >=` 均可用于表达式 |
| 汉语运算符 | `加 减 乘 除 等于 不等于 大于 小于 不少于 不多于` 继续合法 |
| 幂 | `**` 独立右结合语法层，正确处理 `2**3**2` `-2**2` `2**-2` |
| 一元负号 | `负3` 和 `-3` 均为合法，都进入一元运算节点（表层算符不同），`负数量` 是普通名称 |
| `使` 语句 | 仅接受 `加 减 乘 除 变为` 五个汉语变化动作；不接受任何符号运算符；不接受 `整除` `余` |
| `==` | 非法。诊断：「周道使用「=」或「等于」判断相等」 |
| `且/或/并非` | 保留汉语，短路语义继承 Python（但返回操作数还是真/假待 011 裁决） |

### B — 注释与源码保真

| 能力 | 冻结行为 |
|------|---------|
| `注：` 注释 | 持续到行末，不进入运行语义与 Core IR |
| 注释保留 | Parser 不再过滤 COMMENT Token，前导注释和尾行注释在格式化后保留 |
| 定义说明 | 定义前连续注释作为源码说明保留（LSP 悬停候选） |
| 文本转义 | `】】` 表示字面 `】`，`\n` 表示换行 |
| Formatter 注释保真 | 前导注释输出在语句前，尾行注释输出在句号后 |

### C — 结构指称「其」

| 能力 | 冻结行为 |
|------|---------|
| `其` 为关键字 | Lexer 产出 K_ITS；普通用法需 `{其}` 精确名称 |
| 其姓名 | 在遍历体内 → 降低为焦点元素.成员名 |
| 焦点栈 | 语义分析器维护 ITEM_FOCUS 栈，遍历进入时压入，退出时弹出 |
| 嵌套焦点 | 最近焦点优先 |
| 无焦点诊断 | 编译期错误：「其在此处没有可用的当前对象。其只能在遍历主体中使用」 |
| Core IR | 不存在未消解的上下文指称节点（已降级为普通成员访问） |
| 精确名称 | `{其}` `{其姓名}` 绕过关键字，是普通标识符 |

### D — JSON 数据边界

| 能力 | 冻结行为 |

| 解析JSON | 严格解析，拒绝重复键、NaN、Infinity、注释、尾随逗号 |

| 错误定位 | 解析错误含行列位置，生成错误含 JSONPath |

## 3. 正式表层裁决

### 3.1 双表达式表层

以下每对均为正式语法，语义等价：

| 语义 | 汉语式 | 数学符号 |
|------|--------|---------|
| 加法 | `甲加乙` | `甲 + 乙` |
| 减法 | `甲减乙` | `甲 - 乙` |
| 乘法 | `甲乘乙` | `甲 * 乙` |
| 除法 | `甲除乙` | `甲 / 乙` |
| 相等 | `甲等于乙` | `甲 = 乙` |
| 不等 | `甲不等于乙` | `甲 != 乙` |
| 大于 | `甲大于乙` | `甲 > 乙` |
| 小于 | `甲小于乙` | `甲 < 乙` |
| 大于等于 | `甲不少于乙` | `甲 >= 乙` |
| 小于等于 | `甲不多于乙` | `甲 <= 乙` |
| 一元负 | `负3` | `-3` / `-数量` / `-（甲+乙）` |

### 3.2 符号优先（无双表层）

| 语义 | 规范形式 | 状态 |
|------|---------|------|
| 幂 | `甲 ** 乙` | 无双表层 |
| 整除 | `甲 // 乙` | 无双表层 |
| 取余 | `甲 % 乙` | 无双表层 |

### 3.3 历史别名

| 别名 | 状态 | 规则 |
|------|------|------|
| `甲整除乙` | ACCEPTED_LEGACY_ALIAS | 可解析，preserve 保留，symbolic/verbal 转为 `//`，新文档不主动生成 |
| `甲余乙` | ACCEPTED_LEGACY_ALIAS | 同上，转为 `%` |

### 3.4 使语句动词

合法：
```
使数量加1。  使数量减1。  使价格乘折扣。  使总额除人数。  使状态变为【完成】。
```

非法：
```
使数量 + 1。  使数量 ** 2。  使数量 // 2。  使数量 % 2。  使数量整除2。  使数量余2。
```

### 3.5 结构指称

合法：
```
其姓名       ← 遍历焦点领属访问
其地址的城市  ← 焦点链式访问
{其}         ← 精确名称（绕过关键字）
```

非法：
```
显示其。      ← 其不能单独作为值
其的姓名     ← 不采用「其的」形式
显示姓名。    ← 零指代未开放
```

### 3.6 Formatter

```
preserve  （默认）— 保留用户原始写法
verbal    — 统一输出汉语式（加、减、等于……）
symbolic  — 统一输出符号式（+、-、=……）
```

空格规则：汉语式运算符不加空格，符号式二元运算符两侧各一个半角空格，一元负号不加空格。

### 3.7 JSON

外部只使用标准 JSON。周道内部使用映射、列表、文本、整数、小数、真、假、没有值。转换通过显式函数：

```
解析JSON（文本）
生成JSON（值，缩进为2）
```
