## 四、Formatter 反向映射问题

`formatter.py` 第 253-256 行：

```python
映射 = {"+":"加","-":"减","*":"乘","/":"除","//":"整除","%":"余",
        "==":"等于","!=":"不等于",">":"大于","<":"小于",">=":"不少于","<=":"不多于",
        "and":"且","or":"或","in":"在","not_in":"不在","is":"就是"}
```

这意味着：**即使 Lexer/Parser 改为接受符号，Formatter 仍会将所有内部算符字符串重新映射回汉语**。这是必须修正的——目标不是"Formatter 接受符号"，而是"Formatter 输出符号"。

同样，`算术变更` 的 Formatter 映射：

```python
映射 = {"+=": "加", "-=": "减", "*=": "乘", "/=": "除", "//=": "整除", "%=": "余"}
```

`//=` 和 `%=` 尚未审议，不应出现在 Formatter 输出中。

## 五、注释与文本承载追踪

| 功能 | 当前状态 |
|------|---------|
| `注：` 注释 | ✅ Lexer 已支持（行末，`COMMENT` Token） |
| 注释过滤 | ✅ Parser 解析时 `filter(lambda t: t.token_type != COMMENT)` |
| 注释格式化 | ❌ **Formatter 丢弃所有注释**（Parser 过滤后 Formatter 不可见） |
| 定义说明（连续注释） | ❌ **未实现**（注释已被过滤） |
| 文本 `】】` 转义 | ✅ Lexer 已支持 `】】` → 字面 `】` |
| 文本 `\n` 转义 | ✅ Lexer 已支持 |
| SourceMap 注释位置 | ❌ 不可用（注释被过滤） |
| LSP 注释高亮 | ❌ 需确认 |

**根因：Parser 第 108 行过过滤 COMMENT Token** (`self.令牌 = [t for t in self.令牌 if t.token_type != COMMENT]`)。AST 不含注释，Formatter 看不到它们。修复方向：Parser 保留注释作为 Trivia 附加到 AST 节点。

**Formatter 丢失注释是严重问题**——格式化后注释消失，不符合"保真"要求。
