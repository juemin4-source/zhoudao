> 本文件是周道语言的唯一事实源。  
> 所有实现（Parser、Formatter、LSP、AI Quickstart、翻译器、教程）必须以此为准。  
> 规范未记录的结构视为非法，实现不得悄悄接受。

## 总则

### 权威层级

1. 本规范（ZHOUDAO-LANGUAGE-SPEC-1.0）是最高权威
2. 形式语法（zhoudao.ebnf）是本规范的正式表示
3. 测试套件是本规范的可执行验证
4. Parser、Formatter、LSP 的实现行为以本规范为准
5. AI Quickstart、教程、翻译器从本规范抽取子集

### 一致性要求

| 角色 | 要求 |
|------|------|
| Parser | 接受所有正例，拒绝所有非法例 |
| Formatter | 输出规范中指定的唯一推荐形式 |

| 测试 | 覆盖每条产生式的正例、边界例、组合例、非法例 |
| AI/教程 | 只使用规范中"正式"标记的句式 |

## 第一层：词法规范

### 字符集

源码使用 UTF-8 编码。

| 类别 | 范围 |
|------|------|
| 中文字符 | CJK 统一表意文字（一-鿿） |
| ASCII字母 | a-z, A-Z |
| ASCII数字 | 0-9 |
| 下划线 | _ |
| 全角标点 | ， 。 ： ； 、（ ）【】［］《 》 |
| 半角标点 | {} |
| 空白 | 空格、制表符、换行 |

### 名称

名称由中文字符、ASCII 字母、数字和下划线组成，不能以数字开头。

```
标识符 ::= (中文字符 | ASCII字母 | "_") (中文字符 | ASCII字母 | 数字 | "_")*
```

### 精确名称

精确名称由花括号括起，内部为任意名称内容。

```
精确名称 ::= "{" 精确名称内容 "}"
            | "｛" 精确名称内容 "｝"

精确名称内容 ::= (除 "}" "{" 外的任意字符)+
```

**不可违背规则：** 精确名称从左括号到右括号构成一个完整 Token，其内部永不参与关键词、操作符、成员关系或上下文词识别。

### 关键字

关键字分为**硬关键字**和**上下文关键字**。

硬关键字在任何位置都不可作为普通标识符使用：
设、使、为、变为、成立、不成立、如果、就、不然、当、一直、从、中、
每取一项记作、跳出循环、继续下一轮、尝试、如果出错、以、所得、为所得、
引入、显示、定义、设置、类别、包括、包括以下内容、须、不得、否则报错、
删去、就是、不是、本身、报错、依次给出、等待、完成、的所得、可以、
无论是否出错、最后、依、分情形、若为、其余、下文简称、下文所用、
均指全局的、指本定义外层的、默认为、运行如下、规定模块接口、的、
原样报出当前错误、每等到一项记作

上下文关键字只在特定位置具有特殊含义：

| 关键字 | 特殊上下文 | 含义 |
|--------|-----------|------|
| 固定序列 | 后跟 `［` | 固定序列字面量引导 |
| 集合 | 后跟 `［` | 集合字面量引导 |
| 映射 | 后跟 `［` | 映射字面量引导 |
| 错误 | 异常处理分支中 | 当前异常对象 |
| 错误内容 | 异常处理分支中 | 当前异常消息 |
| 自己 | 类别实例方法中 | 当前实例引用 |
| 如下 | 函数/方法定义后 | 定义体引导 |
| 周道源文件 | 引入语句中 | 周道模块来源指示 |
| Python | 引入语句中 | Python 模块来源指示 |

### 字面量

```
整数 ::= 数字+
小数 ::= 数字+ "." 数字+
文本 ::= "【" 文本内容 "】"
布尔 ::= "成立" | "不成立"
空值 ::= "没有值"
```

### 标点

| 符号 | 名称 | 用途 |
|------|------|------|
| `，` | 全角逗号 | 语句/参数分隔 |
| `。` | 句号 | 句子结束 |
| `：` | 全角冒号 | 定义体/接口引导 |
| `；` | 全角分号 | 句内语句分隔 |
| `、` | 顿号 | 列表内分隔 |
| `（` `）` | 全角括号 | 参数列表/表达式分组 |
| `【` `】` | 方头括号 | 字符串字面量 |
| `［` `］` | 空心方括号 | 列表/下标/字面量引导 |
| `《` `》` | 书名号 | 模块名 |
| `{` `}` | 半角花括号 | 精确名称 |

## 第二层：形式语法

完整 EBNF 见 `zhoudao.ebnf`。此处概要：

### 程序结构

```
程序        ::= 顶层定义* 运行入口? EOF

顶层定义     ::= 绑定语句 | 变更语句 | 打印语句
               | 条件语句 | 循环语句 | 遍历语句
               | 尝试语句 | 函数定义 | 类别定义
               | 类别方法定义 | 引入语句 | 模块接口
               | 断言语句 | 作用域声明 | 分情形
               | 空操作 | 删除语句
```

### 运行入口

```
运行入口     ::= "运行如下：" 语句块
```

每篇文章至多一个运行入口，必须位于最后。

### 函数定义

```
函数定义     ::= "定义" 名称 "（" 参数列表? "）"
               ("为" 表达式 "。"
               | "如下：" 语句块)

单表达式形式  ::= "设" 名称 "（" 参数列表? "）" "为" 表达式 "。"
```

### 类别定义

```
类别定义     ::= "设置" 名称 "类别，包括" 字段列表 "。"
字段列表     ::= 字段 ( ("、"|"，") 字段 )*
字段         ::= 名称 (约束)?
```

### 类别方法定义

```
类别方法定义 ::= "定义" 类别名 "类别的" 方法名 "（" 参数列表? "）"
               "如下：" 语句块
```

### 引入语句

```
引入语句     ::= "引入Python模块《" 模块名 "》" 别名?
               | "引入周道源文件《" 模块名 "》" 别名?

从中引入     ::= "从Python模块《" 模块名 "》中引入" 名称列表
               | "从周道源文件《" 模块名 "》中引入" 名称列表

别名         ::= "，下文简称" 名称
```

### 模块接口

```
模块接口     ::= "规定模块接口：" 名称列表
```

### 表达式（概要）

```
表达式       ::= 比较表达式
比较表达式   ::= 加性表达式 (比较算符 加性表达式)?
加性表达式   ::= 乘性表达式 (("加"|"减") 乘性表达式)*
乘性表达式   ::= 前缀表达式 (("乘"|"除"|"整除"|"余") 前缀表达式)*
前缀表达式   ::= "并非（" 表达式 "）" | 后缀起始
后缀起始     ::= 原子 后缀链*
后缀链       ::= "的" 名称 | "【" 文本 "】"
               | "［" 索引 "］" | "（" 参数列表 "）"
```

### 运算符优先级

| 优先级 | 算符 | 结合性 |
|--------|------|--------|
| 最高 | 的（成员访问）、【】（下标）、［］（下标）、（）（调用） | 左 |
| ↑ | 并非 | 右 |
| ↑ | 乘 除 整除 余 | 左 |
| ↑ | 加 减 | 左 |
| ↑ | 大于 小于 不少于 不多于 等于 不等于 在 不在 | 左 |
| ↑ | 且 | 左 |
| 最低 | 或 | 左 |

## 第三层：语义规则（概要）

完整语义规则见 `spec/semantics/`。

### 名称解析

1. 名称从当前作用域向上查找
2. 同一作用域内不同类型可以同名（类别+变量）
3. 精确名称解析为单一 SymbolName，不拆分为成员访问
4. 从 Python 模块引入的名称进入统一周道名称空间

### 执行模式

1. 函数/方法的模式由体内容推导：
   - 包含 `依次给出` 且不含 `等待` → 生成器
   - 包含 `等待` 且不含 `依次给出` → 异步函数
   - 同时包含 → 异步生成器
   - 都不含 → 普通函数
2. 推导由 lowering 阶段完成，语义层不重新推导

### 模块语义

1. 周道模块通过 ModuleLoader 加载，只初始化一次
2. 模块接口列出的名称才能被其他模块选择引入
3. 运行入口在直接执行时运行一次，被引入时不执行
4. 模块缓存以源码内容哈希和编译器版本为键

### 错误语义

1. `报错` 默认引发 RuntimeError
2. 指定错误类型映射到对应 Python 异常类型
3. 分类捕获按源码顺序匹配
4. `原样报出当前错误` 保留异常类型、消息和因果链

## 第四层：规范输出与迁移

### 推荐形式

所有正式语法应输出为 `canonical/canonical-surface-rules.md` 指定的唯一形式。

### 弃用语法

| 弃用形式 | 推荐形式 | 移除计划 |
|---------|---------|---------|
| `元组［…］` | `固定序列［…］` | 012 |
| `引入《甲》` | `引入Python模块《甲》` | 012 |
| `从《甲》中引入乙` | `从Python模块《甲》中引入乙` | 012 |
| `本文公开甲` | `规定模块接口：甲` | 012 |
| `引入周道文《甲》` | `引入周道源文件《甲》` | 012 |
| `设结果为等待甲的所得` | `等待甲完成，记作结果` | 012 |

### 永久非法

- 空接口列表
- 末尾顿号
- 重复模块接口声明
- 运行入口不在文章最后
- 精确名称跨行
