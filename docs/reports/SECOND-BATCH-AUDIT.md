## 2. 新增标点

| 符号 | Token | 作用 |
|------|-------|------|
| ： | `COLON` | 类别清单开启、分情形体开启、定义正文开启 |
| ； | `SEMICOLON` | 类别字段分隔、异常子句边界、分情形分支边界 |

## 3. 分号的三重角色——唯一性审计

分号（；）会在三种结构中均出现，但其上下文可以唯一决定角色：

| 结构 | 前文标记 | 分号含义 |
|------|---------|---------|
| 类别字段 | `包括以下内容：` 之后 | 字段分隔 |
| 异常子句 | `如果出错…就…` 之前后 | try/except/finally 子句边界 |
| 分情形分支 | `依…分情形：` 之后 | 分支分隔 |

审计结论：三种结构由不同的开启短语区分（`包括以下内容：` / `尝试……；` / `依……分情形：`），分号的角色由当前解析的语句类型唯一决定。**不存在歧义。**

## 4. 冒号的角色——唯一性审计

| 结构 | 冒号之前 |
|------|---------|
| 类别清单 | `包括以下内容` |
| 分情形体 | `依…分情形` |
| 定义正文 | 函数名+参数后的逗号？v0.0.1 无冒号 |

v0.0.2 中冒号只出现在三个固定短语之后：`包括以下内容：`、`依…分情形：`。不存在歧义。

## 5. 「类别」的关键字冲突

「类别」在通用中文中是常用词（category），作为关键字可能影响用户选名。

分析：
- Python 的「class」同样影响用户使用「class」作为变量名
- 若用户声明 `设置配置类别，包括名称、值`，表示语义而非数据分类时可用「种类」「类型」替代
- 保留关键字「类别」，用户如有确需可用「类」「种类」「类型」「分类」等近义词
- **不添加转义标识符机制**，与第一批规则一致（grammar.md §12.5）

## 6. 「设置」的分派

```
设置甲类别，包括乙、丙。 → CategoryDecl
设置甲为乙。             → 普通绑定（如第一批「设」）
设置甲类别为【乙】。      → 绑定（「甲类别」是变量名，「为」触发绑定解析）
```

解析策略：
1. `设置` (K_SETUP) 后解析标识符
2. 若标识符后为 `类别` → CategoryDecl
3. 若标识符后为 `为` → 绑定
4. 若标识符后为 `，` → 继续检查后续

## 7. 「就是」与「等于」的边界

| 句式 | 比较内容 | AST 节点 |
|------|---------|---------|
| 甲等于乙 | 内容/数值 | BinOp("==") |
| 甲就是乙 | 对象同一性 | BinOp("is") |
| 甲不是乙本身 | 对象非同一性 | UnaryOp("not", BinOp("is")) |

分别产生不同 AST 节点。

## 8. 「依次给出」与「所得」的边界

| 句式 | 语义 |
|------|------|
| 依次给出甲 | 生成器 yield，不结束定义 |
| 以甲为所得 | return，结束当前定义 |

## 9. 「等待」的作用域限制

- 只能在定义正文内出现
- 顶层出现 → 明确的周道源码错误
- 定义内出现等待 → Emitter 自动输出 async def
- 定义内同时出现等待和依次给出 → 异步生成器

## 10. 「错误内容」的作用域限制

- 只能在 `如果出错，就……` 的范围内使用
- 超出该范围 → 明确的周道源码错误
- 错误 = 异常对象，错误内容 = 异常对象的字符串表示

## 11. 第二批 AST 节点清单

| 节点 | 对应句式 |
|------|---------|
| 导入别名 | 引入《甲》，下文简称乙 |
| 断言 | 甲须/不得满足条件[，否则报错【说明】] |
| 类别声明 | 设置甲类别，包括乙、丙 |
| 类别字段 | 字段名，约束 |
| 删除 | 删去甲 |
| 身份判断 | 甲就是乙 / 甲不是乙本身（表达式） |
| 空操作 | 不作处理 |
| 报错 | 报错【说明】 |
| 依次给出 | 依次给出甲 |
| 等待语句 | 等待甲完成 |
| 等待表达式 | 等待甲的所得 |
| 全局声明 | 下文所用甲，均指全局的甲 |
| 外层声明 | 下文所用甲，指本定义外层的甲 |
| 最终收束 | 无论是否出错，最后甲 |
| 分情形 | 依甲分情形：若为乙，就丙；其余就丁 |
