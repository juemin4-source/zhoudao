# 周道标点作用域规范 (Punctuation Scope)

> **版本**: 0.0.3 — 地基治理
> **关联**: AMBIGUITY-CONSTITUTION.md §6
> **状态**: 冻结

---

## 1. 总则

周道源码使用中文标点作为语法结构标记。每个标点符号在任一语法上下文中最多有一个角色。

---

## 2. 标点符号表

### 2.1 句子终结符

| 符号 | 名称 | Token | 作用 | 上下文限制 |
|------|------|-------|------|-----------|
| 。 | 句号 | PERIOD | 句子终结 | 在任何位置结束当前句子 |
| ！ | 感叹号 | — | 保留（暂未使用） | — |
| ？ | 问号 | — | 保留（暂未使用） | — |

### 2.2 分隔符

| 符号 | 名称 | Token | 作用 | 上下文限制 |
|------|------|-------|------|-----------|
| ， | 逗号 | COMMA | 通用分隔 | 句内表达式、动作链、条件、参数列表 |
| 、 | 顿号 | DUN_HAO | 列表成员分隔 | 列表元素、形参、导入名单 |
| ； | 分号 | SEMICOLON | 结构边界 | 类别字段、异常子句、分情形分支 |
| ： | 冒号 | COLON | 映射/声明 | 类别清单、分情形体、定义正文 |

### 2.3 定界符（配对）

| 符号 | 名称 | Token | 作用 | 配对 |
|------|------|-------|------|------|
| （） | 全角括号 | PAREN_OPEN / PAREN_CLOSE | 表达式分组、函数调用参数、嵌套动作 | 成对 |
| 【】 | 尖头括号 | STRING | 文本字面量 | 成对 |
| ［］ | 全角方括号 | LIST_OPEN / LIST_CLOSE | 列表字面量 | 成对 |
| 《》 | 书名号 | MODULE_OPEN / MODULE_CLOSE | 模块名称 | 成对 |
| {} | 花括号 | EXACT_NAME_OPEN / EXACT_NAME_CLOSE | 精确名称 | 成对 |
| ｛｝ | 全角花括号 | 同上（规范化后） | 精确名称 | 成对 |
| "" | 引号 | — | 保留 | — |
| '' | 引号 | — | 保留 | — |

## 3. 标点的多重角色与消歧

### 3.1 逗号（，）的三重角色

逗号在周道中最为多义，但每种角色由语法上下文唯一确定：

| 角色 | 上下文 | 消歧方式 |
|------|--------|---------|
| 句内顶层表达式分隔 | `表达式, 表达式, 表达式。` | 在解析器 `_解析一句` 中，在则/否/体之后检查 |
| 控制结构内部分隔 | `如果条件，就动作，不然就动作` | 解析器按控制结构规则消费 |
| 动作链分隔 | 定义体/括号分组内 | `_解析动作直到` 内部逗号处理 |

**审计结论**：逗号的三重角色不会导致歧义，因为每种角色由不同的解析函数处理，且解析器向前看一个 token 可确定当前上下文。

#### 3.1.1 逗号消歧示例

```
设甲为1，显示甲。                        ← 逗号分隔两个顶层表达式
如果甲大于0，就显示【正数】。             ← 逗号在条件后
设绝对值（数），如果数不少于0，就以数为所得。 ← 逗号在函数头后（定义体开始）
```

### 3.2 分号（；）的三重角色

| 角色 | 上下文 | 消歧方式 |
|------|--------|---------|
| 类别字段分隔 | `设置类别…包括以下内容：字段；字段` | 仅在类别解析中 |
| 异常子句边界 | `尝试…；如果出错，就…；无论是否出错，最后…` | 仅在尝试解析中 |
| 分情形分支边界 | `依…分情形：若为，就…；其余就…` | 仅在分情形解析中 |

**审计结论**：三种结构由不同的开启短语区分（`包括以下内容：` / `尝试` / `依…分情形：`），分号的角色由当前解析的语句类型唯一决定。

### 3.3 冒号（：）的双重角色

| 角色 | 上下文 | 消歧方式 |
|------|--------|---------|
| 类别清单开启 | `包括以下内容：` 之后 | 前导固定短语 `包括以下内容` |
| 分情形体开启 | `依…分情形：` 之后 | 前导固定短语 `分情形` |
| 定义正文开启 | `定义…如下：` 之后 | 前导固定短语 `如下` |

**审计结论**：冒号前有唯一的固定短语，不存在歧义。

### 3.4 顿号（、）的双重角色

| 角色 | 上下文 | 消歧方式 |
|------|--------|---------|
| 列表元素分隔 | `［甲、乙、丙］` | 在列表解析函数内 |
| 形参分隔 | `定义甲（乙、丙）` | 在形参解析函数内 |
| 导入名分隔 | `从《模块》中引入甲、乙` | 在导入解析函数内 |

**审计结论**：顿号的使用范围由当前解析的语法结构唯一确定。

### 3.5 括号（（））的双重角色

| 角色 | 上下文 | 消歧方式 |
|------|--------|---------|
| 函数调用参数 | `函数名（参数）` | 标识符后 + `（` |
| 表达式分组 | `（表达式）` | 非标识符后 + `（` |
| 嵌套动作分组 | `（动作链）` | 出现在动作位置 |

**审计结论**：`标识符（` → 函数调用；`非标识符（` → 分组；动作位置 `（` → 动作分组。

## 4. 标点配对完整性

### 4.1 配对要求

所有定界符必须成对出现。不配对是编译错误。

```
【文本          → 错误: 文本字面量未闭合
［1、2、3］     → 正确
设置{名称}      → 正确
设（甲          → 错误: 括号未闭合
```

### 4.2 嵌套限制

- 括号 `（）` 允许嵌套：`（（条件））`
- 文本 `【】` 不允许嵌套：`【【内部】】` → 内部 `【` 为普通字符
- 模块名 `《》` 不允许嵌套
- 列表 `［］` 不允许嵌套？允许嵌套表达式列表：`［［1、2］、［3、4］］`
- 花括号 `{}` 不允许嵌套

### 4.3 交叉配对禁止

```
【文本）    → 错误: 交叉配对
```

## 5. 全角/半角统一策略

### 5.1 强制全角的符号

周道的语法性标点**必须使用全角形式**：

| 符号 | 全角（合法） | 半角（非法） |
|------|------------|------------|
| 句号 | 。 | . |
| 逗号 | ， | , |
| 顿号 | 、 | 无对应 |
| 分号 | ； | ; |
| 冒号 | ： | : |
| 括号 | （ ） | ( ) |
| 尖头括号 | 【 】 | [ ]（方括号）|
| 全角方括号 | ［ ］ | [ ] |
| 书名号 | 《 》 | < > |
| 花括号（全角） | ｛ ｝ | { }（见下面）|

### 5.2 半角花括号

半角 `{ }` 也被接受，与全角 `｛｝` 等价：

| 输入 | 规范化后 | 说明 |
|------|---------|------|
| `{名称}` | `｛名称｝` (或维持原样) | 精确名称 |
| `｛名称｝` | `｛名称｝` | 精确名称 |
| `{设}` | `｛设｝` | 关键字转义 |

### 5.3 其余半角字符

半角 `[ ]`、`( )`、`< >` 等其他符号在周道中**均不支持**作为标点使用。

### 5.4 错误纠正建议

当编译器遇到不正确的标点时，应给出明确的纠正建议：

```
[第1行第5列] 错误标点: 使用了半角逗号「,」
建议: 替换为全角逗号「，」
```

```
[第1行第10列] 错误标点: 使用了半角句号「.」
建议: 替换为全角句号「。」
注意: 数字中的小数点（如 3.14）正确为半角 .，不会被此规则影响
```

## 6. 标点上下文的消歧规则总表

| 标点 | 可能角色数 | 是否有歧义 | 消歧依据 |
|------|-----------|-----------|---------|
| 。 | 1 | 否 | 唯一角色 |
| ， | 3 | **否** | 解析器向前看 |
| 、 | 3 | **否** | 当前解析函数 |
| ； | 3 | **否** | 当前语句类型 |
| ： | 3 | **否** | 前导短语 |
| （） | 3 | **否** | 是否在标识符后 |
| 【】 | 1 | 否 | 唯一角色 |
| ［］ | 1 | 否 | 唯一角色 |
| 《》 | 1 | 否 | 唯一角色 |
| {} | 1 | 否 | 唯一角色 |

**结论：周道 v0.0.3 的所有标点在所有语法上下文中都有唯一角色，不存在标点歧义。**

## 7. 句号的作用域

句号（。）是周道中最核心的标点——它定义"句子"的边界。

### 7.1 句子定义

一个句子是一组由逗号分隔的顶层表达式，以句号结束。

```
设甲为1，显示甲。    ← 一个句子，包含两个表达式
设甲为1。显示甲。    ← 两个句子，各包含一个表达式
```

### 7.2 句号的特殊性

句号是唯一在词法层面具有"终结"作用的标点。解析器遇到句号时：

1. 如果当前正在解析动作链/定义体——句号终止整个句子
2. 句号后（非 EOF）开始新句子
3. 句号前未关闭的括号、未匹配的语法结构——错误

### 7.3 句号与数字小数点

数字中的小数点（`.`）不会被误认为句号（`。`）：

```
3.14     → 数字字面量中的半角小数点
3。14    → 错误：句号不能出现在数字间
```

## 8. 冻结清单

| # | 规则 | 说明 |
|---|------|------|
| 1 | 语法标点必须全角 | 强制规则 |
| 2 | 半角 `{}` 和全角 `｛｝` 均接受 | 精确名称标记 |
| 3 | 标点必须配对 | 编译时检查 |
| 4 | 花括号不嵌套 | 精确名称内不能有花括号 |
| 5 | 句号唯一终结作用 | 句子终结 |
| 6 | 逗号多重角色无歧义 | 解析器向前看消歧 |
| 7 | 分号/冒号多重角色无歧义 | 语句类型消歧 |
| 8 | 数字小数点用半角 `.` | 与句号 `。` 区分 |
