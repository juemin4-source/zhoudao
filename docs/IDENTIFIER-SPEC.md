## 1. ExactIdentifier 定义

ExactIdentifier 是周道名称系统的原子单位。每个名称是一个 ExactIdentifier。

### 1.1 字符组成

```
名称字符 := 中文字符 | 英文字母 | 数字 | "_"
名称起始 := 中文字符 | 英文字母 | "_"    （数字不可作首字符）
标点     := 。，、；：（）【】《》［］""''！？…—·　
            ,.;:()[]<>{}""''!?...—· (全角及半角)
```

### 1.2 提取规则

#### 规则 A：普通标识符提取

从当前扫描位置开始，连续读取符合 `名称字符` 的字符序列。遇到以下情况时终止：

1. 遇到任何 `标点` 字符
2. 遇到空白字符（空格、制表符、换行）
3. 当前位置开始匹配到一个**关键字短语**（见保留短语清单）

```
输入: 设用户名称为【张三】。
提取: 设(关键字) → 「用户名」(标识符) → 为(关键字) → 【张三】(文本) → 。(句号)
```

#### 规则 B：花括号精确标识符

`{` 和 `｛` 开启精确名称提取模式：

1. 读取 `{` 或 `｛`
2. 读取内部全部字符（不进行关键字识别，不进行边界切分）
3. 遇到匹配的 `}` 或 `｝` 时终止
4. 花括号内容整体作为标识符名称

```
输入: 设{用户标识码}为100。
提取: 设(关键字) → 「用户标识码」(精确提取) → 为(关键字) → 100(数字) → 。(句号)
```

花括号内禁止：
- 嵌套花括号
- 空内容（`{}` 为非法标识符）
- 纯数字内容

#### 规则 C：关键字冲突标识符

若用户需要的关键字作为变量名，使用花括号精确提取：

```
设{没有值}为【空】。    ← 精确名称「没有值」，不解释为 K_NONE
设{设}为【设置操作】。  ← 精确名称「设」，不解释为 K_SET
```

#### 规则 D：禁止「最长名称优先」

不使用最长名称优先的标识符提取（lexer 的关键字最长匹配是关键字识别，不是名称提取）。

```
输入: 完成任务完成成立
识别: 「完成任务完成」(标识符) → 「成立」(关键字)
禁止: 「完成任务完成成立」整体作为标识符（虽然它更长）
```

### 1.3 ExactIdentifier 类型

```python
@dataclass(frozen=True)
class ExactIdentifier:
    """标识符的精确表示。冻结（不可变）且可哈希。"""
    名称: str          # 名称字符串（规范化后）
    是否精确: bool = False  # True = 花括号提取，False = 普通提取
    源码范围: tuple[int, int] | None = None  # (起始索引, 结束索引)
    
    def __post_init__(self):
        if not self.是否精确:
            # 普通提取的标识符不能是纯关键字
            if self.名称 in 完全关键字集:
                raise 名称错误(f"「{self.名称}」是完全关键字，不可作为标识符", ...)
        # 精确提取的标识符可以是任何非空内容
```

## 2. 名称规范形式

### 2.1 规范化

所有名称在 NameTable 中以规范化形式存储：

1. Unicode NFC 规范化
2. 不区分全角/半角字母（全角英文字母→半角）
3. 保留大小写（中文无大小写，英文区分大小写）
4. 前导/后缀空白在花括号模式中去除

### 2.2 有效名称模式

```
有效名称:
  "用户"
  "用户ID"
  "完成任务"
  "_内部"
  "userName"
  "ABC"
  "版本号2"
  "用户_数量"

非法名称:
  " 用户"          ← 前导空格
  "3个"            ← 数字开头
  "用户-名称"       ← 连字符（非名称字符）
  "用户.名称"       ← 点号
  ""               ← 空名称
  "{名称}"          ← 花括号语法不属名称内容
```

## 3. 保留短语清单

### 3.1 完全关键字（不可作为标识符）

这些短语在任何位置都被识别为关键字，不得作为标识符使用。即使使用花括号精确提取也不能覆盖此限制。

```
设, 使, 为, 变为, 就, 不然，
当, 一直, 从, 中, 每取一项记作，
尝试, 如果出错, 无论是否出错, 最后,
显示, 引入, 设置, 定义,
真, 假, 没有值,
跳出循环, 继续下一轮,
以, 所得, 为所得,
下文所用, 下文简称, 均指全局的, 指本定义外层的,
依, 分情形, 若为, 其余,
还是, 删去,
报错, 不作处理,
决定
```

### 3.2 上下文关键字（由位置决定）

```
成立       — 仅在 设/使 后为关键字；在条件位置参与「命题判断」
不成立     — 同 成立
属于      — 类别字段中可能作为算符
```

### 3.3 保留名称（API）

```
错误       — 异常上下文中被绑定
错误内容   — 异常上下文中被绑定
```

### 3.4 约定保留（非强制、但强烈不建议）

以 `_` 开头的名称：约定为内部名称（类似 Python 的 `_` 前缀）。

## 4. 名称验证规则

### 4.1 注册时验证

当名称进入 NameTable 时，验证：

```
1. 名称非空
2. 名称首字符合法（中文/字母/_）
3. 名称不包含非法字符（标点、运算符、空白）
4. 名称不是完全关键字
5. 名称不在当前作用域的禁止集中
```

### 4.2 运行时验证

```
1. 名称在当前 NameTable 中唯一
2. 名称可以解析到至少一个绑定
3. 名称不模糊（最多一个绑定在同一可见层级）
```

## 5. 名称流边界

### 5.1 关键字边界

当词法扫描器识别到关键字时，该关键字不追加到当前标识符，而是结束上一个标识符并发射关键字 Token。

```
输入: 设用户名称为【张三】
扫描过程:
  '设' → 匹配关键字 K_SET, 发射
  '用' → 开始标识符
  '户' → 继续
  '名' → 继续  
  '称' → 继续
  '为' → 匹配关键字 K_AS, 标识符结束 →「用户名」发射 → 关键字 K_AS 发射
```

### 5.2 标点边界

```
输入: 设年龄为18，显示年龄
扫描过程:
  '设' → K_SET
  '年龄' → 标识符「年龄」
  '为' → K_AS
  '18' → 数字
  '，' → COMMA（标识符边界）
  '显示' → K_PRINT
  '年龄' → 标识符「年龄」
```

### 5.3 花括号边界

```
输入: 设{当前年龄}为18
扫描过程:
  '设' → K_SET
  '{' → EXACT_NAME_OPEN
  '当前年龄' → 标识符「当前年龄」(精确)
  '}' → EXACT_NAME_CLOSE
  '为' → K_AS
  '18' → 数字
```

## 6. 标识符到关键字的冲突消解

当用户代码中标识符与关键字发生冲突时，按以下优先级处理：

### 6.1 完全关键字冲突

```
用户意图: 为变量命名为「设」
代码: 设设为值。       ← 语法错误：第一个「设」是 K_SET，第二个可能被识别为 K_SET
改写: 设{设}为值。     ← 明确：花括号内「设」是精确名称
```

### 6.2 复合关键字冲突

```
用户意图: 为变量命名为「总值」
代码: 设总值为计算。   ← 正确：总(标识符) + 值(标识符) 被识别为「总值」连续标识符
代码: 设总值为100。    ← 正确：「总值」是标识符，然后「为」是 K_AS
```

注意：`总值` 不会与 `为` 冲突，因为 `为` 本身就是关键字，不和 `总值` 连读。

### 6.3 与保留短语的冲突

```
用户意图: 为变量命名为「没有值」
代码: 设没有值为【空】。 ← 歧义：K_SET + K_NONE + K_AS + 文本? 
                          还是 K_SET + 标识符「没有值」+ K_AS + 文本?
解析: 词法层按 KW_SORTED 先匹配到 K_NONE「没有值」，因此解析为前者
改写: 设{没有值}为【空】。← 明确标识符
```

## 7. 名称系统冻结清单

以下关于名称的规则在 v0.0.3 中冻结，变更需走宪法修改流程：

| # | 规则 | 说明 |
|---|------|------|
| 1 | 名称由名称起始+名称字符组成 | 见 §1.1 |
| 2 | 不使用AI分词器 | 名称边界由词法确定 |
| 3 | 不支持最长名称优先 | 名称紧邻关键字时结束 |
| 4 | 花括号强制精确名称 | `{名}` 或 `｛名｝` |
| 5 | 花括号内不识别关键字 | 完全字面提取 |
| 6 | 完全关键字不可为标识符 | 即使使用花括号 |
| 7 | 上下文关键字在特定位置外可为标识符 | 由解析器规则定义 |
| 8 | 名称 NFC 规范化 | 用于去重和查找 |
| 9 | 名称在同一作用域唯一 | 见 NAME-RESOLUTION.md |

## 8. ExactIdentifier Python 实现

```python
@dataclass(frozen=True)
class ExactIdentifier:
    """标识符的精确表示。冻结且可哈希，可用作字典键。"""
    名称: str
    是否精确: bool = False
    源码范围: tuple[int, int] | None = None
    
    def 验证(self, 位置) -> None:
        """验证标识符合法性。"""
        if not self.名称:
            raise 词法错误("空标识符", 位置)
        first = self.名称[0]
        if not (self._是中文字符(first) or first.isalpha() or first == '_'):
            raise 词法错误(f"标识符首字符非法：{first!r}", 位置)
        for ch in self.名称:
            if not (
                self._是中文字符(ch) or ch.isalpha() or ch.isdigit() or ch == '_'
            ):
                raise 词法错误(f"标识符包含非法字符：{ch!r}", 位置)
    
    @staticmethod
    def _是中文字符(ch: str) -> bool:
        return '一' <= ch <= '鿿' or '豈' <= ch <= '﫿'
    
    def __str__(self) -> str:
        return f"ExactIdentifier({self.名称!r}, precise={self.是否精确})"
```
