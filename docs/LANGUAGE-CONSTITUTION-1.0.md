# 周道语言宪法 v1.0

> 本文件是周道语言的正式宪法。宪法确立语言的确定性、分层架构和边界。
> 违反宪法的实现视为缺陷，新增能力不得违反宪法。

---

## 序言：母语内语显化原则

周道以中国使用者理解程序时形成的**母语内语**为表层来源。

中国程序员阅读 Python 代码时，脑内自然将程序结构还原为汉语动作。周道不做"Python 关键词→中文关键词"的机械翻译，而是将这层内语**显化、固定并形式化**。

```
Python 表层
→ Python 实际语义
→ 中国人理解它时的母语内语
→ 消除含糊与省略
→ 固定为周道标准句式
```

### 裁决框架

任何语法争议，不问"像不像中文"，不问"是否兼容 Python"，而问：

> 一个中国开发者真正看懂对应 Python 语义后，脑中会怎样复述这个动作？

例如 `count = 3`，脑内是"设数量为三"而非"对于 count 赋值为三"，因此周道取 `设数量为3。`

### 母语显化率

周道完整度的衡量标准不是"翻译了多少 Python 关键字"，而是：

面对一段程序，使用者是否还需要在脑中继续翻译。

残余的脑内翻译越少，周道越完整。

### 边界

汉化的是关系和动作，保留的是对象身份。

```
引入Python模块《numpy》。  # numpy 是外部对象的正式身份，不翻译
```

第三方 API 名称、Python 标准库名称、开发者定义的变量名，都不在"汉化"范围内。

## 第一章：确定性

### 第1条 精确名称

1. `{名称}` 形式的花括号标识符为精确名称。
2. 精确名称内部永不重新分词。Lexer 输出单一 IDENTIFIER token。
3. 精确名称永不被上下文关键字劫持。
4. Parser 遇到 `是否精确=True` 的 token 时，跳过上下文关键字检查。
5. 精确名称的语义始终是普通名称引用，不因上下文改变。

### 第2条 词法确定性

1. 同一源码每次扫描产生完全相同的 Token 序列。
2. 标识符扫描以关键字边界为中断条件。
3. 不存在后处理阶段对 Token 序列进行重新拆分。
4. Parser 不得通过字符串 `endswith`、`startswith` 等方式猜测词法边界。
5. 上下文关键字具有明确有限前视规则，不进行跨 Token 猜测。

### 第3条 文法确定性

1. 同一合法源码只有一个确定的 Surface AST。
2. 零种解释时报明确的语法错误。
3. 多种解释时报歧义错误并提供候选解析。
4. 不存在 Parser 静默忽略的 Token。

---

## 第二章：编译器分层

### 第4条 分层责任

| 层 | 责任 | 禁止 |
|----|------|------|
| Lexer | Token 序列 | 语义判断 |
| Parser | Surface AST | 名称解析 |
| Lowering | Core IR | 语义检查 |
| Semantic Analyzer | SemanticProgram | 重新分词 |
| Backend | Python AST | 重新语义分析 |

### 第5条 层间契约

1. Surface AST 只描述源码结构，不包含语义信息。
2. Core IR 只描述核心行为，不包含表层句法差异。
3. SemanticProgram 完成名称、作用域、上下文和执行模式确定。
4. PythonAstBackend 不读取 Surface AST。
5. PythonAstBackend 不重新进行名称解析或语义猜测。
6. 正式执行路径不依赖文本 Emitter。

### 第6条 IR 构造契约

1. 所有 IR 节点的新字段必须追加到字段列表末尾。
2. 已有的位置参数构造调用不得因新增字段改变语义。
3. 打破该契约时必须提供兼容构造器和迁移期。

---

## 第三章：语义

### 第7条 名称解析

1. 名称从当前作用域开始向上查找。
2. 精确名称跳过所有上下文关键字判断。
3. 跨模块导入的名称进入统一周道名称空间。
4. Python 模块成员不进入普通周道名称空间。

### 第8条 执行模式

1. 函数和方法的模式（普通/生成器/异步）由体内容推导。
2. 推导在 lowering 阶段完成，语义层不重新推导。
3. 已知异步函数不得被当作普通值使用。
4. 已知生成器不得被当作普通值使用。

---

## 第四章：生态边界

### 第9条 Python 互通

1. Python 模块与周道源文件由源码显式区分。
2. Python API 名称保留原始拼写，不自动翻译。
3. 两种解析路径不得根据文件是否存在互相回退。
4. 环境解析失败必须明确报错，不退避到当前进程。

### 第10条 工具链

1. LSP、Formatter、CLI 和运行器共享同一套编译前端。
2. 不得复制第二套 Parser 或 Semantic Analyzer。
3. 工具链打开文件不得执行用户的 `运行如下`。
4. 缓存策略不得导致 LSP 与 CLI 产生不同语义结果。

---

## 第五章：不变量

1. 无输出不得声称 PASS。
2. 缺 Handler 不得 PASS。
3. `SKIPPED` 不得计为成功。
4. 失败必须留下真实原因。
5. 诊断必须指向周道源码位置。
6. 运行时异常保留原始 Python 类型和因果链。
