> **版本**: 0.0.3 — 地基治理
> **状态**: 冻结
> **前序**: v0.0.2 语法冻结 (ACCEPTANCE-002-R3)
> **关联文档**: IDENTIFIER-SPEC.md, NAME-RESOLUTION.md, CONTEXTUAL-KEYWORDS.md, EXPRESSION-PRECEDENCE.md, PUNCTUATION-SCOPE.md

## 0. 总纲

周道编程语言中，**歧义即错误**（ambiguity is error）。

任何源码结构，凡在无外部上下文的情况下可能产生多于一种解析的，均视为歧义。歧义结构必须在编译时被拒绝，并给出明确的改写建议。

本宪法确立周道名称系统、作用域系统、解析优先级的最高规则。所有后续语法扩展、解析器实现和语义检查均以此宪法为不可绕过的基准。

## 1. 核心原则

### 1.1 一名一指 (One Name, One Reference)

> 每个名称在同一作用域内必须且只能指代一个实体。

同一名称在同一作用域内声明两次，或一个名称指代两个不同实体，均为编译错误。

```
同一域内禁止：
  设甲为1，设甲为2。    → 错误: 同一作用域内重复定义「甲」
```

例外：不同作用域的嵌套不冲突（见作用域规则）。

#### 1.1.1 示例

```zd
设名称为【张三】。       ← 建立「名称」→ "张三"
设名称为【李四】。       ← 错误: 同一作用域内「名称」已定义
```

```zd
设名称为【张三】。       ← 顶层「名称」
定义外层（）如下：
  设名称为【李四】。     ← 允许: 定义「名称」在嵌套作用域
  以名称为所得。          ← 引用嵌套的「名称」
以名称为所得。            ← 引用顶层的「名称」
```

### 1.2 同域不重名 (No Duplicate Names in Scope)

> 同一作用域内声明的名称集合不允许包含重复项。

本规则适用于：
- 变量绑定
- 函数/定义名称
- 类别名称
- 形参名称
- 引入的模块名
- 类别字段名

#### 1.2.1 检测时机

在完成一个作用域的声明的解析后立即检测，不跨作用域后退。

```zd
设甲为1，设甲为2。    → 错误 [重复定义]：「甲」
```

```zd
定义内层（）如下：
  设乙为1，设乙为2。  → 错误 [重复定义]：「乙」
```

```zd
引入《数学》，下文简称工具，
从《文本》中引入工具。  → 错误 [重复名称]：同一域中「工具」已存在
```

### 1.3 省略须唯一 (Omission Must Be Unique)

> 当语言结构允许省略成分时，省略必须唯一可还原，不得存在两个或以上的合法还原。

#### 1.3.1 应用

- `成立` 省略：若 `如果任务完成` 中的 `完成` 可以是变量名也可以是命题后缀，必须唯一。
  - 规则：在 v0.0.3 中，`成立`/`不成立` 在条件位置**不得省略**。明确写作 `如果任务完成成立`。
- 上下文参数省略：若函数调用省略某个参数，该省略必须在签名中唯一匹配。

```zd
如果任务完成成立，就显示【完成】。   ← 正确
如果任务完成，就显示【完成】。       ← 错误: 「完成」是名称还是命题？(v0.0.3 拒绝)
```

### 1.4 结构不得猜测 (Structure Must Not Guess)

> 解析器不得在两个合法解析之间通过"统计"、"习惯"或"常见用法"选择。

#### 1.4.1 禁止行为

- 不因"更常见"而选择某解析路径
- 不使用统计模型选择句法结构
- 不因"用户可能想表达 X"而猜测
- 不使用 AI 或机器学习方法进行消歧

#### 1.4.2 要求

当出现结构化歧义时，必须：
1. 报告歧义错误
2. 列出所有合法解析
3. 给出改写建议

### 1.5 歧义必须报错且给出明确改写 (Ambiguity Must Error With Clear Rewrite)

> 所有可静态检测的歧义必须在编译时报告，并附加至少一种消除歧义的改写。

```zd
设甲为乙。              ← 明确（唯一解析）
如果甲，就显示【乙】。  ← v0.0.3 拒绝 → 「甲」后缺少「成立」「不成立」或条件算符
                          建议: 如果甲成立，就显示【乙】
                          或: 如果甲等于真，就显示【乙】
```

#### 1.5.1 错误格式

```
[第N行第M列] 歧义错误: <描述>
候选解析:
  1. <解析A>
  2. <解析B>
建议改写: <改写说明>
```

### 1.6 花括号只表示精确名称 (Curly Braces for Exact Names Only)

> `{名称}` 或全角 `｛名称｝` 表示"接下来的内容是一个精确的名称标识符，不做分词或上下文猜测"。

这一机制的引入是为了在边界情况下强制指定名称边界，不为语言增加通用表达式能力。

#### 1.6.1 使用场景

花括号不用于通用表达式分组（通用分组使用全角 `（）`）。花括号仅用于名称的精确标记。

```zd
设{用户数量}为100。     ← 「用户数量」是精确名称
设｛用户数量｝为100。   ← 全角版本等同
```

#### 1.6.2 非法使用

```zd
{3加4}                  ← 错误: 花括号内不是有效名称
设{a加b}为c。           ← 错误: 花括号内不是有效名称
```

#### 1.6.3 全角/半角统一

```
半角 {}   — 精确名称符号
全角 ｛｝ — 与半角等价（词法层规范化）
```

## 2. 名称系统

### 2.1 ExactIdentifier

名称的原子单位是 ExactIdentifier。一个标识符必须满足：

1. 以中文字符（CJK Unified Ideographs）、英文字母或 `_` 开头
2. 后续字符可以是中文、英文字母、数字或 `_`
3. 空格不构成名称的一部分
4. 名称边界由分词或花括号声明定义

**不使用 AI/中文分词器/最长名称优先**。标识符的提取由以下严格规则决定：

- 普通位置：连续的中文/英文/数字字符构成一个名称，遇到标点、关键字或空格时结束
- 花括号内：花括号内的全部内容（不含花括号本身）作为精确名称
- 任何被解析为关键字的短语不自动成为标识符

### 2.2 NameTable

NameTable 是一个作用域内的名称 → 实体的映射表。由一个或多个 NameTable 组成的层次结构构成 NameLattice。

见 [IDENTIFIER-SPEC.md](IDENTIFIER-SPEC.md) 和 [NAME-RESOLUTION.md](NAME-RESOLUTION.md)。

### 2.3 名称冻结清单

以下名称被永久冻结为语言保留。用户不得将它们用作标识符：

见 `IDENTIFIER-SPEC.md § 保留短语清单`。

## 3. 作用域

### 3.1 作用域层级

```
全局/模块级 → 定义级（函数） → 控制结构级（如果/循环/尝试）
```

### 3.2 名称搜索规则

1. 当前作用域查找
2. 父作用域查找
3. 模块级全局作用域
4. 显式 `下文所用甲，均指全局的甲` 跨越查找层

见 [NAME-RESOLUTION.md](NAME-RESOLUTION.md)。

## 4. 表达式优先级

完整的算符优先级表见 [EXPRESSION-PRECEDENCE.md](EXPRESSION-PRECEDENCE.md)。

### 4.1 核心规则

```
优先级从低到高：
  或 (or)
  且 (and)
  并非 (not)
  比较 (等于/不等于/大于/小于/不少于/不多于)
  成员 (在…中/不在…中)
  加减 (加/减)
  乘除 (乘/除/整除/余)
  前缀 (负号)
  原子 (数字/文本/列表/标识符/函数调用/括号分组)
```

### 4.2 同一优先级

同一优先级的所有算符左结合。

## 5. 上下文关键字

见 [CONTEXTUAL-KEYWORDS.md](CONTEXTUAL-KEYWORDS.md)。

### 5.1 定义

上下文关键字（contextual keyword）是指只在特定语法位置被视为关键字的短语。在其他位置，它可以作为普通标识符使用。

### 5.2 当前列表

周道 v0.0.3 保留的上下文关键字：

| 关键字 | 上下文 | 作为标识符 |
|--------|--------|-----------|
| 成立 | 设/使/如果 后 | 禁止（位置歧义） |
| 不成立 | 设/使/如果 后 | 禁止（位置歧义） |
| 所得 | 以…为所得 中 | 允许（需消歧） |
| 最后 | 无论是否出错 后 | 允许（需消歧） |
| 以下内容 | 包括 后 | 允许 |

完全关键字（始终不可作为标识符）：

| 关键字 | 原因 |
|--------|------|
| 设, 使, 如果, 当, 从, 就, 不然, 一直 | 句法基石 |
| 尝试, 跳出循环, 继续下一轮 | 控制结构 |
| 显示, 引入, 设置, 定义 | 操作关键字 |
| 没有值 | 空值占用 |

## 7. 歧义分类

### 7.1 词法歧义

两个或以上的词法切分方式能匹配同一源码段。

```
例: 「没有值」是单个 K_NONE 还是标识符「没有」+ 标识符「值」?
规则: KW_SORTED 最长匹配优先
后果: 仍歧义时 -> 词法歧义错误
```

### 7.2 语法歧义

同一 token 序列可以匹配两个或以上的语法产生式。

```
例: 设标识符没有值 是 空值绑定 还是 绑定 + 表达式?
规则: 由句法位置唯一区分
```

### 7.3 名称歧义

同一名称指代两个或以上的目标。

```
例: 同一作用域内「名称」同时为变量和类别
规则: 同域不重名
```

### 7.4 作用域歧义

名称在当前作用域中不可见或存在多个可见候选。

```
例: 嵌套定义中的名称引用——当前域无定义，父域有定义
规则: 向父作用域查找，找到唯一的最近定义
```

### 7.5 省略歧义

省略导致还原选项 > 1。

```
例: 如果甲 —— 甲是命题还是变量名？
规则: v0.0.3 拒绝省略，要求显式「成立」/「不成立」
```

## 8. 歧义处置流程

```
源码
  ↓
词法扫描（最长关键字匹配）
  ↓ (若匹配冲突 → 词法歧义错误)
语法解析（递归下降 + 向前看1）
  ↓ (若解析冲突 → 语法歧义错误)
名称检查（NameTable 构建 + 名称绑定）
  ↓ (若名称冲突 → 名称错误)
作用域解析（NameLattice 查找）
  ↓ (若查找冲突 → 作用域错误)
表达式类型检查
  ↓
发射 / 执行
```

### 8.1 歧义错误等级

| 等级 | 类型 | 处理方式 |
|------|------|---------|
| L0 | 词法歧义 | 编译时错误，不可恢复 |
| L1 | 语法歧义 | 编译时错误，不可恢复 |
| L2 | 名称歧义 | 编译时错误，不可恢复 |
| L3 | 作用域歧义 | 编译时错误，不可恢复 |
| L4 | 省略歧义 | 编译时错误，不可恢复 |

所有等级均不降级为警告。

## 9. 本宪法的修改

本宪法一经冻结，修改必须满足：

1. 有至少 3 个不同源的示例表明当前规则造成不可用的限制
2. 修改方案不得减少歧义覆盖范围
3. 修改方案必须经过审计委员会审查
4. 修改后所有 200+ ambiguity_cases 测试必须通过
