## 1. NameTable

NameTable 是单一作用域内名称→绑定的映射表。

### 1.1 定义

```python
@dataclass
class NameTable:
    """单一作用域的名称映射表"""
    作用域类型: str  # "global" | "function" | "control" | "category" | "match"
    父表: "NameTable | None" = None
    _表: dict[str, 绑定信息] = field(default_factory=dict)
    
    def 绑定(self, 名称: str, 实体: 绑定信息) -> None:
        if 名称 in self._表:
            raise 语义错误(f"重复定义：「{名称}」已在当前作用域中定义")
        self._表[名称] = 实体
    
    def 查找(self, 名称: str) -> "绑定信息 | None":
        return self._表.get(名称)
    
    @property
    def 所有名称(self) -> frozenset[str]:
        return frozenset(self._表.keys())
```

### 1.2 绑定信息

```python
@dataclass
class 绑定信息:
    名称: str
    类型: str  # "变量" | "函数" | "类别" | "模块" | "参数"
    位置: "源码位置"
    是否可重绑定: bool = False  # 由「使」引入的名称
```

### 1.3 作用域类型

| 类型 | 创建时机 | 备注 |
|------|---------|------|
| global | 模块级 | 整个 .zd 文件的顶层 |
| function | 定义/设函数 时 | 每个定义创建新作用域 |
| control | 如果/当/遍历/尝试 体 | 控制结构体内部 |
| category | 类别声明 | 类别字段命名空间 |
| match | 分情形 | 分支内避免名称污染 |

## 2. NameLattice

NameLattice 是由 NameTable 组成的层次结构，表示完整程序的作用域图。

### 2.1 定义

```python
@dataclass
class NameLattice:
    """完整程序的作用域层次结构"""
    根表: NameTable
    当前表: NameTable
    
    def 进入作用域(self, 类型: str) -> NameTable:
        新表 = NameTable(作用域类型=类型, 父表=self.当前表)
        self.当前表 = 新表
        return 新表
    
    def 离开作用域(self) -> NameTable:
        旧表 = self.当前表
        self.当前表 = 旧表.父表 or self.根表
        return 旧表
    
    def 解析(self, 名称: str, 位置) -> "绑定信息":
        """从当前作用域向上查找名称。"""
        表 = self.当前表
        visited = set()
        while 表 is not None:
            id(表)  # 避免循环引用（安全防护）
            绑定 = 表.查找(名称)
            if 绑定 is not None:
                return 绑定
            表 = 表.父表
        raise 语义错误(f"未定义的名称：「{名称}」", 位置)
    
    def 在当前域解析(self, 名称: str) -> "绑定信息 | None":
        """仅当前表查找，不递归到父表。"""
        return self.当前表.查找(名称)
    
    def 检查重复(self, 名称: str) -> bool:
        """当前作用域中是否有同名绑定。"""
        return self.当前表.查找(名称) is not None
```

### 2.2 解析优先级

名称解析在 NameLattice 中按以下优先级：

```
当前作用域变量/函数 > 当前作用域参数 > 父作用域 > 全局
```

### 2.3 解析失败

当名称不可解析时，编译器必须：

1. 报告具体的名称
2. 报告查找路径（已搜索的作用域）
3. 如果名称类似已有名称，提示候选

```
[第3行第5列] 未定义的名称：「数量」
已搜索作用域:
  - 定义「计算总和」(L2)
  - 全局 (L1)
最接近的候选名称：「总数」「数量表」「数」
```

## 3. 作用域规则

### 3.1 模块级（全局）作用域

- 模块级声明的名称对整个模块可见
- `设` 在模块级创建全局名称
- `定义` 在模块级创建全局函数名
- `设置 类别` 在模块级创建类别名
- 模块级名称对嵌套定义可见（除非被覆盖）

```zd
设全局变量为100。          ← 全局作用域
定义读取全局（）如下：
  显示全局变量。            ← 正确：可见
```

### 3.2 定义级（函数）作用域

- 每次 `定义` 或 `设…（…）为…` / `设…（…），…，以…为所得` 创建一个新作用域
- 形参在该作用域中绑定
- 函数体内部的声明在该作用域中

```zd
定义计算（数量）如下：       ← 函数作用域开始
  设加倍为数量乘2。         ← 「加倍」在函数作用域
  以加倍为所得。            ← 引用「加倍」
                             函数作用域结束
显示加倍。                  ← 错误：「加倍」不在当前作用域
```

### 3.3 控制结构级作用域

- `如果`/`当`/`遍历`/`尝试` 的体创建控制结构级作用域
- 在该作用域内 `设` 的名称不会泄漏到外部

```zd
如果条件成立，就
  设临时值为100。          ← 控制结构作用域
  显示临时值。              ← 正确
显示临时值。                ← 错误：控制结构作用域已退出
```

注意：当前周道 v0.0.2 实现中，控制结构体不创建独立作用域（Python 的 `=` 赋值会影响外部）。v0.0.3 歧义宪法**建议但暂不强制**控制结构级作用域隔离。

### 3.4 嵌套作用域与遮蔽

子作用域可以声明与父作用域同名的名称，但不建议这样做（可能引起混淆）：

```zd
定义外层（）如下：
  设名称为【外层】。
  定义内层（）如下：
    设名称为【内层】。      ← 遮蔽外层的「名称」
    显示名称。              ← 显示「内层」
  显示名称。                ← 显示「外层」
```

### 3.5 显式作用域声明

周道提供显式作用域跨越机制：

```zd
设全局配置为【默认】。
定义读取配置（）如下：
  下文所用全局配置，均指全局的全局配置。  ← 显式绑定到全局
  设全局配置为【覆盖】。                  ← 操作的是全局变量
```

```zd
定义计数器工厂（初始值）如下：
  设当前值为初始值。
  定义递增（）如下：
    下文所用当前值，指本定义外层的当前值。  ← nonlocal 声明
    使当前值加1。
    以当前值为所得。
```

## 4. 名称搜索算法

### 4.1 标准查找

```
function 查找(名称, 起始表):
    当前 = 起始表
    while 当前 is not None:
        if 名称 in 当前.表:
            return 当前.表[名称]
        当前 = 当前.父表
    return NOT_FOUND
```

### 4.2 全局跨越查找

当 `下文所用甲，均指全局的甲` 声明后，对该名称的查找直接从全局表开始：

```
function 全局查找(名称, 起始表):
    全局 = 起始表.找根()
    if 名称 in 全局.表:
        return 全局.表[名称]
    return NOT_FOUND
```

### 4.3 外层跨越查找

当 `下文所用甲，指本定义外层的甲` 声明后，查找跳过当前定义层，从外层定义开始：

```
function 外层查找(名称, 起始表):
    # 找到最近的 function 类型作用域并跳转到它的父表
    当前 = 起始表.父表
    while 当前 is not None and 当前.作用域类型 != "function":
        当前 = 当前.父表
    if 当前 is None:
        return NOT_FOUND
    return 查找(名称, 当前.父表)  # 从函数的外部继续标准查找
```

## 5. 引用的时序

### 5.1 前向引用

周道中，在同一作用域内不能使用尚未声明的名称。

```zd
设结果为甲加1。            ← 错误：「甲」尚未定义
设甲为100。
```

```zd
设甲为100。               ← 正确：先声明
设结果为甲加1。            ← 正确：后使用
```

### 5.2 函数前向引用

函数定义可以互相引用（因为函数体在执行时才求值），但定义本身的名称必须在调用点前可见：

```zd
定义甲（）如下：           ← 正确：定义先于调用
  显示【甲】。
定义乙（）如下：
  甲（）。                 ← 正确：调用时「甲」已在全局表中
```

```zd
乙（）。                   ← 错误：调用时「乙」尚未声明
定义乙（）如下：
  显示【乙】。
```

## 6. 类别字段名称解析

### 6.1 类别声明中的名称

类别声明本身创建一个类别名，其字段名在类别命名空间中唯一：

```zd
设置用户类别，包括以下内容：
  名称，须为文本。
  年龄，须为整数。         ← 「名称」和「年龄」是字段名
```

### 6.2 字段名冲突

```zd
设置用户类别，包括以下内容：
  名称，须为文本。
  名称，须为整数。         ← 错误：类别「用户」包含重复字段「名称」
```

## 7. 名称解析错误

### 7.1 错误类型总表

| 错误类型 | 触发条件 | 示例 |
|---------|---------|------|
| 重复定义 | 同域重名 | `设甲为1，设甲为2。` |
| 未定义 | 引用了未声明的名称 | `显示乙。` 且无 `乙` |
| 遮蔽冲突 | 显式声明冲突 | 下文所用 和 本地 同时声明 |
| 访问越界 | 超出作用域访问 | 在定义外引用定义内变量 |
| 跨越冲突 | 同一名称多重跨越 | 同时全局和外层声明 |

### 7.2 错误消息格式

```
[第N行第M列] <错误类型>：<描述>
  作用域路径：<当前域 → 父域 → 全局>
  建议：<改写建议>
```

## 8. 冻结清单

| # | 规则 | 说明 |
|---|------|------|
| 1 | NameTable 每个作用域唯一 | 一个作用域一个表 |
| 2 | 解析从当前表向上查找 | 找到最近的定义 |
| 3 | 同域不重名 | 违反即编译错误 |
| 4 | 前向引用禁止（变量） | 变量使用前必须声明 |
| 5 | 函数前向引用允许 | 函数体在调用时求值 |
| 6 | 显式跨越需声明 | 全局/外层跨越必须写 |
| 7 | 类别字段名隔离 | 字段在其类别命名空间内 |
| 8 | 删除时解除绑定 | `删去` 从当前表移除 |
