# Python 元数据策略规范 v0.0.10

> 本规范描述周道如何读取和使用 Python 类型元数据。
> 元数据用于补全、悬停和参数诊断，不影响运行时行为。

---

## 1. 配置模式

```toml
[lsp]
python_metadata = "stubs_only"  # 默认值
```

### 三档定义

| 模式 | 行为 |
|------|------|
| `off` | 不读取任何 Python 元数据；仅提供模块名和周道自身信息 |
| `stubs_only` | 只读取 .pyi、typeshed 和已安装包的自带存根；不导入模块 |
| `stubs_then_safe_inspect` | 优先存根；存根缺失时有限安全 inspect；失败静默降级为"未知" |

### 选择建议

- 保守项目：`stubs_only`（默认）
- 开发体验优先：`stubs_then_safe_inspect`

---

## 2. 来源标注

每条元数据必须标明来源：

| 标注 | 含义 |
|------|------|
| `来自存根` | 由 .pyi 或 typeshed 提供 |
| `来自运行时` | 由安全 inspect 获取 |
| `未知` | 无法确定 |

---

## 3. 安全规则

- 补全不得导入会产生明显副作用的模块
- inspect 超时或失败必须静默降级
- 任何 Python 元数据失败不得导致 LSP 崩溃
- 元数据缺失时不得伪造确定结论
