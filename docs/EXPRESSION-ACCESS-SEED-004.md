> **版本**: 0.0.4 — 表达式取用
> **状态**: 草案（Phase 004-01）
> **前序**: v0.0.3 歧义宪法冻结 (AMBIGUITY-CONSTITUTION.md)
> **关联**: AMBIGUITY-CONSTITUTION.md, IDENTIFIER-SPEC.md, EXPRESSION-PRECEDENCE.md, PUNCTUATION-SCOPE.md

## 0. 总纲

本规范定义周道的**统一表达式取用系统**。

### 0.1 解决的问题

在 v0.0.3 及之前，周道可以：
- 绑定名称（`设甲为乙`）
- 变更名称（`使甲变为乙`）
- 删除名称（`删去甲`）
- 调用函数（`函数名（参数）`）

但无法：
- 取得对象中的成员（如 `对象.属性`）
- 取得集合中的项目（如 `列表［索引］`）
- 调用对象上的方法（如 `对象.方法（参数）`）
- 建立和操作键值映射（如 dict）
- 对成员和项目进行统一的读取、变更与删除

### 0.2 核心设计

```text
的   → 成员取得运算符（等价 Python 的 .）
【】  → 字符串键下标（表达式后使用时）
［］  → 表达式下标（表达式后使用时）
设 → 仅建立名称绑定
使 → 改变已存在的名称/项目/成员
删去 → 删除名称、项目或成员（统一删除语句）
```

### 0.3 不变规则

- `{}` / `｛｝` 永久只表示精确名称（v0.0.3 冻结）
- `设` 只建立名称绑定，不变更已有名称
- `使` 改变已经存在的名称/项目/成员
- `删去` 可以作用于名称、项目或成员，不设计第二套删除语句
- 成员取得与 Python 成员中文别名映射必须分层，Parser 不得硬编码具体成员名
- 任何合法表达式只能产生唯一 AST
- 不得依靠中文概率分词、词频或大模型消歧

## 1. 成员取得运算符 `的`

### 1.1 语法

```
成员访问  := 表达式 "的" 标识符
```

`的` 是周道的成员取得运算符，对应 Python 的 `.`（属性访问）。

### 1.2 语义

```zd
设甲为乙的丙。           → 乙.丙
显示甲的乙。             → print(甲.乙)
使甲的乙变为丙。          → 甲.乙 = 丙
```

`的` 读取对象的**命名成员**。在 `的` 左侧可以是任意表达式（变量、函数调用结果、其他成员访问等），右侧必须是标识符。

### 1.3 连续成员访问

`的` 具有左结合性，连续使用构成成员访问链：

```zd
甲的乙的丙          → 甲.乙.丙
甲的乙的丙的丁       → 甲.乙.丙.丁
```

### 1.4 优先级

`的` 的优先级**高于函数调用**，低于所有字面量/原子：

```text
等级 8 (成员访问)  : 的                     （最高绑定力，立即取得）
等级 7 (后缀)      : 函数调用 下标
等级 6 (原子)      : 字面量、标识符、括号分组
...
```

设计理由：
- `甲的乙（丙）` → `(甲的乙)(丙)` = `甲.乙(丙)` — 先取得成员，再调用其结果
- 这是方法调用的自然语义：`对象.方法(参数)`

### 1.5 语法对比

| 周道 | Python | 说明 |
|------|--------|------|
| `甲的乙` | `甲.乙` | 成员读取 |
| `甲的乙（丙）` | `甲.乙(丙)` | 方法调用 |
| `甲的乙的丙` | `甲.乙.丙` | 链式成员 |
| `甲的乙（丙）的丁` | `甲.乙(丙).丁` | 方法结果再取成员 |

### 1.6 用于 `设` 和 `使`

`的` 在 `设` 之后**只能读取**（获取成员值），不能通过 `设` 创建成员绑定：
- `设甲的乙为丙。` → `设` 建立名称 `甲` 或 `甲的乙`，语义为：将 `甲的乙` 的值绑定给该名称
- `使甲的乙变为丙。` → 改变 `甲的乙` 的值，要求 `甲` 是可变对象

注：`设甲的乙为丙` 中的 `甲的乙` **不是新建成员赋值**，而是读取成员值作为表达式。如果要改变成员，使用 `使`。

```zd
设甲为字典。             ← 建立名称
使甲的【键名】变为值。     ← 改变字典中的项目
设乙为甲的【键名】。       ← 读取项目，绑定给乙
删去甲的【键名】。         ← 删除字典中的项目
```

## 2. 下标访问

### 2.1 字符串键下标 `【键】`

```
字符串键下标 := 表达式 "【" 内容 "】"
```

`【内容】` 在表达式之后使用时，表示**以字符串作为键的下标访问**。

```zd
映射【键名】    → 映射["键名"]
甲的乙【丙】    → 甲.乙["丙"]
```

当 `【】` 出现在表达式开头（或运算符之后），它仍是字符串字面量：

```zd
显示【测试】    → print("测试")    （字符串字面量，非下标）
设甲为【值】    → 甲 = "值"         （字符串字面量，非下标）
```

消歧规则：
- `【】` 在表达式起始位置或运算符之后 → 字符串字面量
- `【】` 紧跟在表达式之后 → 字符串键下标

### 2.2 表达式下标 `［］`

```
表达式下标 := 表达式 "［" 表达式 "］"
```

全角方括号 `［］` 用于下标访问时，括号内是通用表达式：

```zd
列表［索引］    → 列表[索引]
列表［0］       → 列表[0]
列表［索引加1］ → 列表[索引 + 1]
```

### 2.3 下标类型总结

| 形式 | 含义 | Python 等效 |
|------|------|-------------|
| `甲【键】` | 字符串键下标 | `甲["键"]` |
| `甲［乙］` | 表达式下标 | `甲[乙]` |
| `甲［0］` | 整数下标 | `甲[0]` |

### 2.4 下标作为操作目标

下标访问的结果可以作为读取、变更和删除的目标：

```zd
设值为列表［0］。            ← 读取
使列表［0］变为新值。         ← 变更
删去列表［0］。               ← 删除
使字典【键名】变为新值。       ← 字典变更
删去字典【键名】。             ← 字典删除
```

## 3. 后缀操作链

### 3.1 统一后缀处理

表达式解析后，可以连续应用后缀操作：

```
后缀链 := （"的" 标识符 | "【" 内容 "】" | "［" 表达式 "］" | "（" 参数 "）"）*
```

所有后缀操作在解析时连续应用，遵循以下顺序：
1. `的` 成员访问 → 左结合
2. `【】` 字符串键下标 → 左结合
3. `［］` 表达式下标 → 左结合
4. `（）` 函数调用 → 左结合

### 3.2 后缀链示例

```zd
甲的乙              → 成员访问: 甲 → .乙
甲的乙（丙）        → 成员访问 → 函数调用: 甲 → .乙 → (丙)
甲的乙【丙】        → 成员访问 → 字符串下标: 甲 → .乙 → ["丙"]
甲的乙（丙）［丁］  → 成员访问 → 函数调用 → 下标: 甲 → .乙 → (丙) → [丁]
```

### 3.3 后缀组合规则

所有后缀操作在同一层级平坦组合，不涉及优先级嵌套——它们全部在后缀位置按出现顺序依次应用。

```
3 的 成员 【 键 】（ 参数 ）
→ 从 3 开始
→ 3.成员          (的访问)
→ 3.成员["键"]    (字符串下标)
→ 3.成员["键"](参数) (函数调用)
```

## 4. 方法调用

### 4.1 通过 `的` 调用方法

方法调用是 `的` + `（）` 的联合使用：

```zd
甲的乙（丙）    → 甲.乙(丙)
```

这等价于：
1. 通过 `的` 取得 `甲` 的 `乙` 成员
2. 对取得的成员进行函数调用

### 4.2 链式方法调用

```zd
甲的乙（丙）的丁（戊）
→ 甲.乙(丙).丁(戊)
```

### 4.3 成员名中文别名映射

成员名的中文→Python 映射由独立的 `emitter.py` 负责，Parser 不硬编码具体成员名。

映射采用分层结构：
```python
_成员映射 = {
    "随机": {
        "随机整数": "randint",
        "随机选择": "choice",
    },
    "__内置__": {
        "绝对值": "abs",
        "长度": "len",
    },
}
```

- `从《随机》中引入随机整数` → `from random import randint as 随机整数`
- `从《列表》中引入排序` → 映射到 `sorted`
- Parser/lexer 完全不感知这些映射

## 5. 映射（字典）操作

### 5.1 映射字面量

映射字面量使用 `【键：值】` 格式，多个键值对用 `、` 或 `，` 分隔：

```text
映射 := "【" 键值对（"、" 键值对）* "】"
         | "【" 键值对（"，" 键值对）* "】"

键值对 := 键 "：" 值
键    := 文本字面量 | 标识符
值     := 表达式
```

由于 `【】` 被用作字符串字面量和下标，映射字面量需要一个不同的语法。

**方案**：在表达式解析器中，当 `【】` 出现在表达式起始位置，且内容中包含 `：` 时，解析为映射字面量；否则解析为字符串字面量。

> 注意：此方案存在"内容中含 `：` 的字符串"被误判的风险。
> 因为在 v0.0 系列中暂不实现映射字面量，由 Python dict 在运行时通过映射提供。
>
> 当前阶段（SEED-004）暂不引入映射字面量新语法，使用 Python 原生 `dict()` 或通过 `的设置` 来创建映射。

### 5.2 运行时映射操作

用户可以通过 Python 标准映射接口操作字典：

```zd
从【内置】中引入字典。
设映射为字典（）。
使映射的【键】变为值。
删去映射的【键】。
显示映射的【键】。
```

映射的下标操作由 Python 运行时提供。周道提供一致的 `的` 成员访问和 `【】`/`［］` 下标操作。

## 6. 统一读取、变更与删除

### 6.1 读取

读取通过三种方式实现：

| 方式 | 语法 | 目标 |
|------|------|------|
| 名称引用 | `甲` | 名称 |
| 成员访问 | `甲的乙` | 成员 |
| 下标访问 | `甲【键】` / `甲［0］` | 项目 |

任何表达式在运算位置（如 `显示` 的参数、`如果` 的条件、绑定的右边）均可求值。

### 6.2 变更

变更统一使用 `使`：

```zd
使甲变为值。              ← 变更名称
使甲的乙变为值。           ← 变更成员
使甲【键】变为值。         ← 变更项目（字符串键）
使甲［0］变为值。          ← 变更项目（表达式下标）
```

算术变更同样支持：

```zd
使甲加1。                 ← 名称算术变更
使甲的乙加1。             ← 成员算术变更
使甲［0］加1。            ← 项目算术变更
```

### 6.3 删除

删除统一使用 `删去`：

```zd
删去甲。                  ← 删除名称
删去甲的乙。              ← 删除成员
删去甲【键】。            ← 删除项目（字符串键）
删去甲［0］。             ← 删除项目（表达式下标）
```

`删去` 后面跟随一个表达式（表示要删除的目标）。解析器确保目标可以被删除：
- 名称 → `del 名称`
- 成员 → `del 名称.成员`
- 项目 → `del 名称["键"]` 或 `del 名称[索引]`

## 7. 完整 EBNF 语法

### 7.1 程序结构（新增部分）

```
程序        := 句子+
句子        := 顶层表达式（"，" 顶层表达式）* "。"

顶层表达式  := 绑定
            | 布尔绑定
            | 变更
            | 算术变更
            | 布尔变更
            | 输出
            | 单式函数
            | 复式函数
            | 引入
            | 从中引入
            | 如果句
            | 当句
            | 遍历句
            | 尝试句
            | 跳出
            | 继续
            | 删除           （增强：支持成员和项目删除）
```

### 7.2 语句（增强部分）

```
绑定        := "设" 表达式 "为" 表达式      （增强：右侧可为任意表达式）
            | "设" 标识符 "为" 表达式

变更        := "使" 左值 "变为" 表达式
算术变更     := "使" 左值 算术算符 表达式
布尔变更     := "使" 左值 "成立"
            | "使" 左值 "不成立"
            
删除        := "删去" 左值

左值         := 标识符
             | 成员访问
             | 字符串键下标
             | 表达式下标

以…为所得    := "以" 表达式 "为所得"
```

### 7.3 后缀操作（新增）

```
后缀链       := （后缀操作）*

后缀操作     := 成员访问
            | 函数调用
            | 字符串键下标
            | 表达式下标

成员访问     := "的" 标识符

函数调用     := "（" （表达式（"，" 表达式）*）？ "）"

字符串键下标 := "【" 内容 "】"        （当紧跟在表达式之后时）

表达式下标   := "［" 条件 "］"
```

### 7.4 表达式层级

```
条件        := 或表达式
或表达式    := 且表达式（"或" 且表达式）*
且表达式    := 非表达式（"且" 非表达式）*
非表达式    := "并非" "（" 条件 "）"
            | 比较表达式
比较表达式  := 成员表达式（比较算符 成员表达式）？
            | 成员表达式（身份算符 成员表达式）？
            | 成员表达式（空值后缀）？
成员表达式  := 加性表达式（成员后缀）？
            | 加性表达式（空值后缀）？
加性表达式  := 乘性表达式（加性算符 乘性表达式）*
乘性表达式  := 前缀表达式（乘性算符 前缀表达式）*
前缀表达式  := 算符？ 后缀起始
后缀起始    := 原子 后缀链*

原子        := 数字
            | 文本              （字符串字面量）
            | 列表              （列表字面量）
            | 真值
            | 假值
            | 空值
            | 标识符
            | "（" 条件 "）"     （括号分组）
            | "【" 键值对列表 "】"  （映射字面量，未来扩展）

文本        := "【" [^】：]* "】"   （不含：的为字符串字面量）
```

注：后缀链紧随原子（或分组后的任何表达式），在解析器中后缀操作连续应用，不允许前缀或二元运算符插入在后缀链中间。

### 7.5 左值

```
左值         := 标识符 后缀链
```

任何合法的 `标识符 + 后缀链` 都可以作为左值（读取、变更、删除的目标）。

例外：
- 函数调用不能作为左值（`甲() = 乙` 无意义）
- 只有变量的最终引用可以作为变更/删除目标

## 8. 解析器变更

### 8.1 后缀操作解析

在 `_解析原子()` 之后增加后缀解析循环：

```python
def _解析后缀链(self, left):
    """解析后缀操作链并应用到 left。"""
    while True:
        tok = self._当前()
        if tok.token_type == K_DE:      # 的 → 成员访问
            self._吃()
            member = self._期望(IDENTIFIER)
            left = 成员访问(left, member.值)
        elif tok.token_type == STRING and self._上次是表达式:
            # 【内容】在表达式后 → 字符串键下标
            key = self._吃().值
            left = 字符串下标(left, key)
        elif tok.token_type == LIST_OPEN:
            # ［内容］ → 表达式下标
            left = self._解析表达式下标(left)
        elif tok.token_type == PAREN_OPEN:
            # （参数）→ 函数调用
            left = self._解析调用续(left)
        else:
            break
    return left
```

### 8.2 左值解析

`设`、`使`、`删去` 后跟的左值可以是带后缀链的表达式：

```python
def _解析左值(self):
    """解析可以作为赋值/删除目标的左值。"""
    名称令牌 = self._期望(IDENTIFIER)
    left = 变量(名称令牌.值)
    # 解析后缀链
    result = self._解析后缀链(left)
    # 检查：最终结果必须是变量引用（不是函数调用或纯字面量）
    return result
```

### 8.3 对字符串字面量的消歧

`【内容】` 的双重角色（字符串字面量 vs 字符串键下标）通过解析上下文区分：

1. 在表达式解析的**起始**位置 → 字符串字面量（现有行为不变）
2. **紧跟在表达式之后** → 字符串键下标

判定规则：
- `_解析原子()` 中遇到 STRING token → 字符串字面量（仅当没有已解析的左值）
- `_解析后缀链()` 中遇到 STRING token → 字符串键下标（有已解析的左值）

## 10. 发射器变更

### 10.1 成员访问发射

```python
def _发射成员访问(self, 节点: 成员访问) -> str:
    return f"{self._发射表达式(节点.对象)}.{节点.成员}"

def _发射字符串下标(self, 节点: 字符串下标) -> str:
    return f"{self._发射表达式(节点.对象)}[\"{节点.键}\"]"

def _发射表达式下标(self, 节点: 表达式下标) -> str:
    return f"{self._发射表达式(节点.对象)}[{self._发射表达式(节点.索引)}]"
```

### 10.2 删除发射

```python
def _发射删除(self, 节点: 删除) -> str:
    target = self._发射表达式(节点.目标)
    self._添加行(f"del {target}")
```

### 10.3 变更发射

当变更为成员访问或下标时，生成对应的赋值：

```python
def _发射变更(self, 节点: 变更) -> str:
    self._添加行(f"{self._发射表达式(节点.目标)} = {self._发射表达式(节点.值)}")
```

这与现有逻辑一致——现有代码已经为 `变更` 生成 `name = value`。Enhanced 版本只需 `目标.表达式` 即可正确生成 `obj.attr = value` 或 `obj["key"] = value`。

### 10.4 方法调用发射

方法调用是 `成员访问` + `调用` 的组合：

```python
def _发射调用(self, 节点: 调用) -> str:
    func = self._发射表达式(节点.函数)
    args = ", ".join(self._发射表达式(p) for p in 节点.参数)
    return f"{func}({args})"
```

当 `节点.函数` 是 `成员访问` 时，自然生成 `对象.方法(参数)`。

## 11. 映射分层与 Parser 隔离

### 11.1 分层架构

```text
Lexer / Parser               ← 完全不知晓成员映射
    ↓
AST（成员访问 / 调用）
    ↓
Emitter                     ← 成员映射在此层
    ↓
Python 代码
```

### 11.2 Emitter 映射表

成员名中文→Python 映射存在于 `emitter.py` 的 `_成员映射` 字典中。

Parser 生成的 AST 使用**中文成员名**。Emitter 在发射代码时进行映射：

```python
def _发射表达式(self, 节点) -> str:
    ...
    if isinstance(节点, 成员访问):
        member = 节点.成员
        # 在已知模块范围内查找映射
        py_name = self._查找成员映射(节点.对象, member)
        return f"{self._发射表达式(节点.对象)}.{py_name}"
```

Parser 完全不感知这些映射。新增成员映射只需修改 `emitter.py` 的 `_成员映射` 字典。

## 14. 未实现（Non-Goals for v0.0.4）

1. **复合赋值表达式**：`甲的乙加1` 在表达式中作为值返回（Java/C 风格）——暂不实现
2. **映射字面量新语法**：暂不引入 `【k:v】` 映射字面量，使用 `dict()` 替代
3. **嵌套解构**：从成员访问结果中解构绑定
4. **默认成员**：`toString`、`__dict__` 等魔术成员的自动映射
5. **链式比较的恢复**：`a < b < c` 仍保持错误
6. **Null-safe 成员访问**：`甲?.乙` 语法不引入
7. **切片操作**：`甲［开始：结束］` 语法暂不引入
8. **属性装饰器**：`@property` 等效语法

## 15. 歧义审计

### 15.1 `的` 的潜在歧义

| 表达式 | 解析 | 歧义？ |
|--------|------|--------|
| `甲的乙` | 成员访问 | ❌ 唯一解析 |
| `甲的乙的丙` | 链式访问 | ❌ 唯一解析 |
| `甲的乙（丙）` | 成员后调用 | ❌ 唯一解析（后缀顺序定义） |
| `甲（的乙）` | 括号分组 | ❌ 括号优先级最高，`的乙` 是括号内 |

### 15.2 `【】` 的双重角色

| 表达式 | 解析 | 歧义？ |
|--------|------|--------|
| `【测试】`（表达式起始） | 字符串字面量 | ❌ 由位置唯一确定 |
| `甲【键】` | 字符串键下标 | ❌ 由位置唯一确定 |
| `显示【测试】` | 字符串字面量 | ❌ `显示` 是关键字，`【测试】` 在表达式起始 |
| `甲（【键】）` | 括号 + 字符串 | ❌ `【键】` 在括号内，是表达式起始 |

### 15.3 `的` 与精确名称 `{}`

| 表达式 | 解析 | 歧义？ |
|--------|------|--------|
| `{甲的乙}` | 精确名称「甲的乙」 | ❌ `{}` 永久为精确名称 |
| `甲的{乙}` | 语法错误 | ❌ `的` 后必须跟标识符，不能跟 `{` |
| `${乙}` | 不适用 | ❌ `$` 不是合法字符 |

## 17. 冻结清单

| # | 规则 | 说明 |
|---|------|------|
| 1 | `的` 是成员访问运算符 | 对应 Python `.` |
| 2 | `【】` 在表达式后是字符串键下标 | 在表达式起始是字符串字面量 |
| 3 | `［］` 是表达式下标 | 对应 Python `[]` |
| 4 | `{}` 永久只表示精确名称 | v0.0.3 冻结，不可更改 |
| 5 | `设` 只建立名称绑定 | 不变更已有名称/项目/成员 |
| 6 | `使` 改变左值 | 左值可以是名称/成员/项目 |
| 7 | `删去` 删除左值 | 不设计第二套删除语句 |
| 8 | 后缀链左结合 | 连续后缀按顺序应用 |
| 9 | `的` 优先级高于函数调用 | `甲的乙（丙）` = `(甲的乙)(丙)` |
| 10 | Parser 不硬编码成员名 | 成员映射在 Emitter 层 |
| 11 | 任何表达式只能产生唯一 AST | 无句法歧义 |
| 12 | 所有已有测试必须通过 | 向后兼容 |
