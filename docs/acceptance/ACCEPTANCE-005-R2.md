## 一、交付能力

### 1. 精确名称 `{}`

花括号 `{}` / `｛｝` 内部整体形成一个精确名称。内部的「的、加、减、时、完成」等关键字不得重新参与分词，保护内部语义关键词不被解析为语法元素。

```zd
设{用户的姓名}为【张三】。                          → 用户的姓名 = "张三"
显示{用户的姓名}。                                    → print(用户的姓名)
显示{用户的资料}的年龄。                              → print(用户的资料.年龄)
设{时加完成}为True。                                  → 时加完成 = True
```

**对比**（同名不加花括号退化为成员访问）：
```zd
设{用户的姓名}为【整体】。    → 用户的姓名 = "整体"  （单一标识符）
使用户的姓名变为【拆分】。    → 用户.姓名 = "拆分"    （成员访问，的→.）
```

**语义规则**：
- 花括号内部整体形成单一标识符，不参与分词
- 半角 `{}` 与全角 `｛｝` 语义相同
- 格式化输出统一使用半角 `{}`
- 禁止空名称、禁止跨行、禁止嵌套花括号

### 2. 映射字面量 `映射［键为值］`

`映射` 关键字作上下文关键字，后跟 `［键为值、键为值］` 形成映射字面量：

```zd
设m为映射［【x】为10、【y】为20］。    执行 → 键为字符串的 dict
设m为映射［结果为基数加5］。          值可以是表达式  
映射［【键】为1、键2为值2］          混合字符串键与变量键
```

**语义规则**：
- `映射` 仅后跟 `［` 时触发映射字面量解析，否则为普通标识符
- 映射降低为现有 `映射字面量IR` 节点，不创建第二套映射节点
- 裸 `［键为值］`（不带 `映射`）不被当作映射字面量
- 重复静态字面量键按 004 规范诊断

### 3.1 新增产生式

```ebnf

标识符内容     := （非花括号、非换行字符）+

映射字面量     := "映射" "［" 映射条目（"，" 映射条目）* "］"
              | "［" """" 表达式 "为" 表达式 """"（"，" """ 表达式 "为" 表达式 """）* "］"
```

| 文件 | 变更 |

## 八、已知限制

1. **精确名称 `{}` 只保护「的、加、减、时、完成」等关键词**：它不创建新作用域，花括号内也不支持表达式——只是名称字面量保护语法。
2. **映射字面量键类型**：`映射［键为值］` 的键可以是任意表达式（不限于字符串）。如果是变量键，生成的 Python dict 以该变量值为键。
3. **上下文关键字 `映射`**：`映射` 在当前实现中仅当后跟 `［` 时才触发映射字面量解析。作为普通标识符使用时行为正常。但若标识符 `映射` 后意外跟 `［`（如 `设映射［甲］为乙。`），则会被解释为映射字面量而非期望的列表下标。
4. **空映射**：`映射［］` 产生空 dict `{}`，但当前解析器不提供专门的空映射错误提示（会尝试解析然后遇到 `］` 而结束）。
5. **映射中的精确名称**：`映射［{键}为值］` 中 `{键}` 作为变量名引用，不会触发花括号内关键词重新分词。
