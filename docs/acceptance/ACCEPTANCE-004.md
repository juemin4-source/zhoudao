## 一、交付能力

### 1. 成员取得运算符 `的`

```zd
设名称为用户的姓名。             → 用户.姓名
设内容为文本的strip（）的split（）。 → 文本.strip().split()
设结果为记录的get（【等级】）。      → 记录.get("等级")
```

`的` 是后缀链中的成员访问运算符，优先级高于函数调用，左结合。

### 2. 字符串键下标 `【】`

```zd
使用户【年龄】变为4。             → 用户["年龄"] = 4
删去用户【年龄】。                → del 用户["年龄"]
显示映射【姓名】。                → print(映射["姓名"])
```

`【内容】` 在表达式起始位置为字符串字面量（兼容 v0.0.3），在表达式之后为字符串键下标。

### 3. 表达式下标 `［］`

```zd
设首项为记录们［0］。             → 记录们[0]
设末项为记录们［0减1］。          → 记录们[0 - 1]
使列表［0］变为100。             → 列表[0] = 100
删去列表［1］。                  → del 列表[1]
```

### 4. 切片下标 `［开始：结束］`

```zd
设开头为文本［0：3］。            → 文本[0:3]
显示文本［：3］。                → print(文本[:3])
显示文本［0：］。                → print(文本[0:])
显示文本［：］。                  → print(文本[:])
```

### 5. 映射字面量 `［【键】为【值】］`

```zd
设用户为［【姓名】为【周道】］。     → 用户 = {"姓名": "周道"}
设映射为［【姓名】为【张三】、【年龄】为25］。
→ 映射 = {"姓名": "张三", "年龄": 25}
```

通过 2-token 向前看区分列表与映射：第一个元素后跟 `为` 则为映射，否则为列表。

### 6. 统一变更目标

`使` 可以作用于名称、成员和项目：

```zd
使数量变为5。                    → 数量 = 5             （名称）
使用户的年龄变为30。              → 用户.年龄 = 30       （成员）
使映射【键】变为新值。             → 映射["键"] = 新值    （项目）
使列表［0］变为100。              → 列表[0] = 100        （表达式下标）
使用户的年龄加5。                 → 用户.年龄 += 5       （算术变更）
```

### 7. 统一删除目标

`删去` 可以作用于名称、成员和项目：

```zd
删去甲。                         → del 甲               （名称）
删去用户的名称。                  → del 用户.名称         （成员）
删去映射【键】。                  → del 映射["键"]        （项目）
删去列表［1］。                   → del 列表[1]           （表达式下标）
```

## 三、语法变更

### 3.1 新增产生式

```ebnf
后缀链       := （后缀操作）*
后缀操作     := "的" 标识符            （成员访问）
              | "【" 内容 "】"          （字符串键下标）
              | "［" 条件 "］"          （表达式下标）
              | "［" 开始 "：" 结束 "］" （切片下标）
              | "（" 参数 "）"           （函数调用）

映射字面量   := "［" 映射条目（"，" 映射条目）* "］"
映射条目     := 表达式 "为" 表达式

左值         := 标识符 后缀链*
```

### 3.2 新增 AST 节点

| 节点 | 字段 | 用途 |
|------|------|------|
| `成员访问` | `对象: 表达式, 成员: str` | 对应 `的` |
| `字符串下标` | `对象: 表达式, 键: str` | 对应 `【】` |
| `表达式下标` | `对象: 表达式, 索引: 表达式` | 对应 `［］` |
| `切片下标` | `对象: 表达式, 开始, 结束` | 对应 `［：］` |
| `映射字面量` | `条目: list[tuple[表达式, 表达式]]` | 对应 `［【】为【】］` |

## 四、关键设计决策

### 4.1 `的` 为词法关键字

`的` 加入 KW_MAP，词法层作为单字符关键字匹配。最长匹配机制确保 `的所得`（K_OF_RESULT）等已有多字符关键字不受影响。

### 4.2 `【】` 消歧规则

`【内容】` 在表达式**起始**位置为字符串字面量（兼容 v0.0.3），在表达式**之后**为字符串键下标。消歧由解析器位置确定——不会误判。

### 4.3 `［］` 列表/映射消歧

`［］` 内部内容通过 2-token 向前看区分：
- `［元素1 为 值1］` → 映射字面量
- `［元素1、元素2］` → 列表字面量
- `［］` → 空列表

### 4.4 Parser/成员名映射分层

Parser 生成含中文成员名的 `成员访问` 节点。Emitter 负责中文→Python 成员名映射（通过 `_成员映射` 字典），Parser 不硬编码任何成员名。

## 六、冻结清单 (v0.0.4)

| # | 规则 | 说明 |
|---|------|------|
| 1 | `的` 是成员访问运算符 | 后缀链中处理，优先级高于函数调用 |
| 2 | `【内容】` 表达式后为下标 | 表达式起始为字符串字面量 |

| 7 | `设` 只建立名称绑定 | 不变更已有名称/项目/成员 |
| 8 | `使` 改变左值 | 左值可以是名称/成员/项目 |
| 9 | `删去` 删除左值 | 不设计第二套删除语句 |
| 10 | 后缀链左结合 | 连续后缀按顺序应用 |
| 11 | Parser 不硬编码成员名 | 成员映射在 Emitter 层 |
| 12 | 任何表达式只产生唯一 AST | 无句法歧义 |
| 13 | 所有已有测试必须通过 | 向后兼容 |

## 九、已知限制

1. **映射字面量键类型**：当前映射字面量的键可以是任意表达式（不限于字符串）。如果键是非字符串类型，生成的 Python dict 可以处理。
2. **运行时类型约束**：`使` 和 `删去` 对成员/项目的操作可能因 Python 对象类型限制而在运行时失败（语法上正确）。
3. **链式方法调用**：`文本的strip（）的split（）` 等链式调用已支持，但成员名中文→Python 映射需 Emitter 层配置。
4. **切片步长**：`［开始：结束：步长］` 三步长切片暂不实现。
