## 1. 上下文关键字定义

上下文关键字（contextual keyword）是指仅在特定语法位置被解释为关键字的短语。在非关键字位置，它可以作为普通标识符。

这一机制的目的是在保留足够关键字数量的同时，不过度消耗自然中文词汇。

---

## 2. 上下文关键字分类

### 2.1 命令上下文关键字

这些短语只在 `设` / `使` / `设置` 等命令关键字后担任关键字角色：

| 短语 | 关键字类型 | 激活上下文 | 作为标识符的可能 |
|------|-----------|-----------|-----------------|
| 成立 | K_TRUE_STATE | `设 名 成立` / `使 名 成立` 中的第二个位置 | 理论上可能，但强不建议（歧义） |
| 不成立 | K_FALSE_STATE | `设 名 不成立` / `使 名 不成立` 中的第二个位置 | 同上 |
| 没有值 | K_NONE | `设 名 没有值` / 条件后缀 | 同上。若用户坚决需要，用花括号 |
| 为 | K_AS | `设 名 为 表达式` | 理论上可能，但完全关键字化 |
| 变为 | K_BECOME | `使 名 变为 表达式` | 同上 |

### 2.2 条件上下文关键字

这些短语只在条件表达式中有关键字语义：

| 短语 | 关键字类型 | 激活上下文 | 作为标识符 |
|------|-----------|-----------|-----------|
| 等于 | OP_EQ | 比较表达式 | 允许（如 `设等于为运算名`） |
| 不等于 | OP_NE | 比较表达式 | 允许 |
| 大于 | OP_GT | 比较表达式 | 允许 |
| 小于 | OP_LT | 比较表达式 | 允许 |
| 不少于 | OP_GE | 比较表达式 | 允许 |
| 不多于 | OP_LE | 比较表达式 | 允许 |
| 在 | OP_IN | 成员表达式 | 允许 |
| 不在 | OP_NOT_IN | 成员表达式 | 允许 |
| 且 | OP_AND | 逻辑表达式 | 允许 |
| 或 | OP_OR | 逻辑表达式 | 允许 |
| 并非 | OP_NOT | 逻辑表达式（前缀） | 允许 |

### 2.3 控制结构上下文关键字

这些短语只在特定控制结构的特定位置有关键字含义：

| 短语 | 关键字类型 | 激活上下文 | 绑定位置 |
|------|-----------|-----------|---------|
| 所得 | K_RESULT | `以 表达式 为所得` / `… 的所得` | 必须紧跟在 `为` 或 `的` 后 |
| 为所得 | K_AS_RESULT | `以 表达式 为所得` | 必须紧跟在表达式后 |
| 如果出错 | K_EXCEPT | `尝试 … 如果出错` | 尝试体右侧 |
| 最后 | K_FINALLY_DO | `无论是否出错，最后` | 无论是否出错 右侧 |
| 每取一项记作 | K_EACH_AS | `从 … 中，每取一项记作` | 固定短语 |
| 下文简称 | K_AS_ALIAS | `引入 模块，下文简称` | 引入右侧 |
| 下文所用 | K_SCOPE_DECL | 语句开头 | 独立语句 |
| 均指全局的 | K_GLOBAL | `下文所用 … 均指全局的` | 作用域声明 |
| 指本定义外层的 | K_NONLOCAL | `下文所用 … 指本定义外层的` | 作用域声明 |
| 若为 | K_CASE | `依 … 分情形：若为` | 分支开头 |
| 其余 | K_DEFAULT | `依 … 分情形：… 其余` | 分支开头 |

### 2.4 完全上下文解析的关键字

这些短语在词法层就被区分，不需要语法上下文：

| 短语 | 关键字 | 原因 |
|------|--------|------|
| 无论是否出错 | K_FINALLY | 6字固定短语，唯一识别 |
| 包括以下内容 | K_INCLUDE_LONG | 6字固定短语 |
| 继续下一轮 | K_CONTINUE | 4字固定短语 |
| 跳出循环 | K_BREAK | 4字固定短语 |
| 依次给出 | K_YIELD | 4字固定短语 |

## 3. 上下文关键字消歧算法

### 3.1 词法层消歧

词法层（lexer）使用**最长匹配优先**识别固定短语。固定短语总是优先于其子串：

```
输入: 继续下一轮测试
词法: K_CONTINUE("继续下一轮") + IDENTIFIER("测试")
      而不是 「继续」+「下一轮」+「测试」
规则: KW_SORTED 中「继续下一轮」比「继续」长，优先匹配
```

### 3.2 语法层消歧

语法层（parser）使用**向前看**（lookahead）来区分同一短语的不同角色：

```
场景: 「没有值」在 设 后
向前看: 设 标识符 → 如果下一个是「没有值」→ 空值绑定
        设 标识符 → 如果下一个是「为」 → 普通绑定
场景: 「没有值」在 条件 中
向前看: 表达式 → 如果当前是「没有值」→ 空值后缀判断
```

### 3.3 歧义冲突解决方案

当上下文关键字可能被误解时：

1. **完全关键字化**: 如果某短语在任何语义角色中都有关键字含义（如`设`、`使`、`就`），则完全关键字化——永远不可作为标识符
2. **花括号转义**: 若要使用关键字短语作为标识符，使用 `{短语}` 语法
3. **改写建议**: 若编译器检测到可能混淆，给出改写建议

## 4. 关键字冲突决策表

以下是用户意图（行）与词法解析结果（列）的对照：

### 4.1 设 上下文

| 用户写 | 解析结果 | 正确? |
|--------|---------|-------|
| `设甲为乙` | K_SET + ID("甲") + K_AS + ID("乙") | ✓ |
| `设甲成立` | K_SET + ID("甲") + K_TRUE_STATE | ✓ |
| `设成立为真` | K_SET + IDENTIFIER? → 词法层 `设`(K_SET) + `成立` → 是 K_TRUE_STATE 还是 IDENTIFIER? | 歧义! |
| `设{成立}为真` | K_SET + ID("成立") + K_AS + LIT_TRUE | ✓ 花括号强制 |

### 4.2 条件上下文

| 用户写 | 解析结果 | 正确? |
|--------|---------|-------|
| `如果年龄大于18` | K_IF + 表达式(年龄 > 18) | ✓ |
| `设大于为运算符` | K_SET + ID("大于") + K_AS + ID("运算符") | ✓（作为标识符） |
| `如果(年龄大于)就` | K_IF + 表达式(年龄 > ???) | 歧义：缺少右操作数 → 语法错误 |

### 4.3 具有双角色的短语

| 短语 | 作为关键字 | 作为标识符 | 区分方式 |
|------|-----------|-----------|---------|
| 成立 | `设甲成立` / `如果甲成立` | 标识符位置 | 位置：在标识符后且无 `为` 时是关键字 |
| 不成立 | `设甲不成立` | 标识符位置 | 同上 |
| 没有值 | `设甲没有值` / `甲没有值` | 标识符位置 | 位置 + 花括号转义 |
| 所得 | `以甲为所得` | `设所得为值` | 在 `为` 后是关键字部分 |
| 最后 | `最后显示结束` | `设最后为值` | 在 `无论是否出错` 后是关键字 |

## 5. 关键字在语法高亮中的处理

### 5.1 完全关键字

完全关键字应始终以关键字颜色高亮（如蓝色/紫色）：

```
设, 使, 如果, 当, 从, 就, 不然, 一直, 尝试, 显示, 引入, 
设置, 定义, 真, 假, 没有值, 跳出循环, 继续下一轮
```

### 5.2 上下文关键字

上下文关键字应在相应的语法上下文中以关键字颜色高亮，非关键字上下文中以标识符颜色高亮。

上下文关键字高亮需要语法感知编辑器支持。

### 5.3 条件算符

算符类型的关键字应使用算符颜色（如红色）：

```
等于, 不等于, 大于, 小于, 不少于, 不多于, 且, 或, 并非, 在, 不在
```

## 6. 关键字的添加纪律

### 6.1 添加条件

新的关键字只有在以下条件下才能添加：

1. 有至少 5 个真实用例表明缺少该关键字导致不可接受的代码复杂度
2. 该关键字不与现有标识符产生无法消解的冲突
3. 有完整的歧义测试用例
4. 通过宪法审计

### 6.2 禁止添加

- 过于通用的单字（如 `的`、`了`、`是`、`有`）— 除非有极端理由
- 英文/拼音关键字（如 `def`、`class`、`for`、`return`）
- 与现有完全关键字同义的关键字

### 6.3 完全关键字 vs 上下文关键字的决策

| 考虑因素 | 倾向完全关键字 | 倾向上下文关键字 |
|---------|--------------|----------------|
| 使用频率 | 极高（每个程序都出现） | 中等 |
| 语义特异性 | 唯一语义（如 `设`） | 多语义可能（如 `成立`） |
| 与自然语言冲突 | 低（如 `显示`） | 高（如 `最后`） |
| 被用作标识符的需求 | 低 | 高 |

## 7. 冻结清单

| # | 规则 | 说明 |
|---|------|------|
| 1 | 完全关键字不可覆盖 | 即使在花括号内 |
| 2 | 上下文关键字在非激活位置是标识符 | 但建议避免使用 |
| 3 | 短语在词法层优先识别 | 最长匹配 |
| 4 | 语法层向前看分辨率 | 标准递归下降 |
| 5 | 花括号可转义上下文关键字 | `{关键字}` 变为标识符 |
| 6 | 条件算符都是上下文关键字 | 可用作标识符 |
| 7 | 单字中文不轻易加入关键字 | 需要极端理由 |
