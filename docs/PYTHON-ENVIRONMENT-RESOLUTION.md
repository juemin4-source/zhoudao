# Python 环境解析规范 v0.0.10

> 本规范描述周道项目中 Python 环境的解析方式。
> 环境解析必须确定、可解释、可复现，不得依靠隐蔽的 sys.path 偶然性。

---

## 1. 解释器选择

### 1.1 确定方式

Python 解释器按以下优先级确定：

1. 项目配置 `[python].interpreter`（绝对路径或相对项目根）
2. 当前虚拟环境（`$VIRTUAL_ENV` 或 `.venv/bin/python`）
3. 系统默认 `python3`

### 1.2 规则

- 解释器路径必须可执行
- 不得自动搜索并选择多个可能的虚拟环境
- 缺少配置时允许单文件模式，但必须报告使用的解释器

### 1.3 环境报告

每次解析必须输出：

- 解释器绝对路径
- Python 版本
- sys.path 列表
- 路径来源（项目配置 / 虚拟环境 / 系统默认）

---

## 2. 搜索路径

### 2.1 来源

Python 搜索路径来源：

| 来源 | 说明 |
|------|------|
| 选定解释器的 `sys.path` | 由目标解释器进程返回 |
| 项目 `extra_paths` | 配置中显式声明的附加路径 |
| 当前工作目录 | 仅单文件模式且未配置时 |

### 2.2 失败处理

- 解释器不可执行 → 明确报错，不退避到当前进程
- `sys.path` 获取失败 → 明确报错
- 环境解析失败 → CLI 返回退出码 3

---

## 3. 贯穿规则

- CLI、LSP、运行器和测试必须使用相同的环境解析结果
- 环境解析结果必须缓存，同一进程内重复使用
- 环境报告可导出为 JSON 供 CI 使用
